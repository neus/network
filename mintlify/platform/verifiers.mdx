Verifiers define what you prove. The API catalog is code-first: backend `verifierRegistry.js` is the single source of truth; SDKs, widgets, and MCP consume it via `GET /api/v1/verification/verifiers`.

Verifiers are the building blocks of NEUS. Each verifier proves something specific—wallet ownership, domain control, NFT holding, human verification, and more.

## How Verifiers Work

| Type | What It Proves | Flow |
|------|----------------|------|
| **Ownership** | Wallet, domain, NFT, token | Instant or lookup |
| **Identity** | Human verification, org membership | Interactive |
| **Risk** | Wallet safety, bot resistance | Lookup |
| **Agents** | AI identity, delegation | Instant |

## Get the Live Catalog

```http
GET /api/v1/verification/verifiers
```

The API returns all verifiers, required fields, and metadata. Use it as the source of truth—SDKs, widgets, and MCP consume the same catalog.
