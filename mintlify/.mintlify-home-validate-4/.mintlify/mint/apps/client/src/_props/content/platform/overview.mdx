# Platform Overview

NEUS is a verification and trust layer built around one loop:

1. Verify
2. Receive a proof receipt
3. Reuse that receipt for access, trust, monetization, and agent policy

## Design principles

* One hosted verify path, not fragmented auth and checkout routes.
* One backend verifier catalog, not duplicated metadata maps across frontend, SDK, MCP, and docs.
* One proof lifecycle, reused everywhere instead of re-verifying ad hoc.
* Private by default, with explicit public and discoverable modes when needed.
