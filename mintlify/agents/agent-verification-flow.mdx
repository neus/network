NEUS agent integrations usually combine three layers:

1. Agent identity proof
2. Optional controller-to-agent delegation proof
3. An execution surface such as MCP that can read verification results or trigger hosted verification

## Recommended Sequence

<Steps>
  <Step title="Create an agent identity proof">
    Bind the agent wallet and agent metadata into a reusable proof receipt.
  </Step>
  <Step title="Create an agent delegation proof when a controller is authorizing the agent">
    Use delegation to define scope, permissions, and spend or disclosure constraints.
  </Step>
  <Step title="Use MCP or API surfaces to consume the verification results">
    Agents can inspect verifier catalog data, run gate checks, and hand users to hosted verify for interactive completion.
  </Step>
</Steps>

## Hosted Agent Link

When `neus_agent_link` (MCP) returns `status: "link_required"`, the user is sent to hosted verify to complete agent identity and delegation. The URL can include:

| Param | Use |
|-------|-----|
| `preset=agent-pack` | Preload agent-identity, agent-delegation, wallet-risk |
| `agent=<wallet>` | Pre-fill agent wallet when linking |

Example: `https://neus.network/verify?preset=agent-pack&agent=0x...`
