# Choose an Integration Path

NEUS supports four primary integration surfaces. Start with the surface that matches your product constraints, then expand into additional surfaces as your trust model evolves.

## Pick by job to be done

<CardGroup cols={2}>
  <Card title="Ship a browser or app flow" icon="code" href="../sdks/overview">
    Use the JavaScript SDK when you need proof creation, status polling, hosted verify handoff, or private proof access in a custom UI.
  </Card>
  <Card title="Gate content quickly" icon="shield" href="../widgets/overview">
    Use Widgets when you want the fastest hosted or embedded path with React components and script-tag badge rendering.
  </Card>
  <Card title="Integrate AI or agent tooling" icon="cpu" href="../cookbook/mcp">
    Use the MCP server when an AI client needs verifier lookup, gate checks, or a hosted verify handoff without reimplementing backend logic.
  </Card>
  <Card title="Go deeper on backend policy" icon="terminal" href="../api/overview">
    Use the HTTP API when your backend needs gate checks, verifier catalog reads, auth code exchange, or explicit status polling.
  </Card>
</CardGroup>

## Recommended journey

<Steps>
  <Step title="Start with reader intent">
    Decide whether you need onboarding, access gating, proof issuance, hosted checkout, or agent-facing trust signals.
  </Step>
  <Step title="Choose the thinnest viable surface">
    Prefer Widgets for the fastest launch, the SDK for custom UX, the API for backend policy, and MCP for AI-native integrations.
  </Step>
  <Step title="Keep `/verify` as the canonical hosted path">
    Interactive flows should converge on the hosted verify surface instead of inventing parallel auth or checkout routes.
  </Step>
  <Step title="Treat proof receipts as reusable trust artifacts">
    Status, access, and downstream policy should key off the same proof receipt rather than duplicating verification logic in every surface.
  </Step>
</Steps>

## Which path should you choose?

| If you need... | Start here | Then add |
|---|---|---|
| Fastest production launch | [Widgets](../widgets/overview) | [SDK](../sdks/overview) for custom UX |
| Custom browser UX | [SDK](../sdks/overview) | [API](../api/overview) for backend checks |
| Backend-only eligibility decisions | [API](../api/overview) | [Cookbook](../cookbook/overview) for rollout patterns |
| AI assistant or agent workflows | [MCP](../cookbook/mcp) | [Agents](../agents/overview) for proof models |
| Org/app onboarding and sponsored flows | [Hub Integrator Setup](../cookbook/hub-integrator) | [Auth + Hosted Verify](../cookbook/auth-hosted-verify) |
