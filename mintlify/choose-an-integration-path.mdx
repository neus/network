NEUS supports four integration surfaces. Start with the one that matches your needs, then add others as required.

## Pick by Use Case

<CardGroup cols={2}>
  <Card title="Ship a browser or app flow" icon="code" href="./sdks/overview">
    Use the JavaScript SDK when you need proof creation, status polling, hosted verify handoff, or private proof access in a custom UI.
  </Card>
  <Card title="Gate content quickly" icon="shield" href="./widgets/overview">
    Use Widgets when you want the fastest hosted or embedded path with React components and script-tag badge rendering.
  </Card>
  <Card title="Integrate AI or agent tooling" icon="cpu" href="./cookbook/mcp">
    Use the MCP server when an AI client needs verifier lookup, gate checks, or hosted verify handoff.
  </Card>
  <Card title="Server-side policy enforcement" icon="terminal" href="./api/overview">
    Use the HTTP API for gate checks, verifier catalog reads, auth code exchange, or status polling from your server.
  </Card>
</CardGroup>

## Integration Steps

<Steps>
  <Step title="Define your requirements">
    Identify what you need: onboarding, access gating, proof issuance, hosted checkout, or agent verification.
  </Step>
  <Step title="Choose the minimal surface that fits">
    Use Widgets for fastest launch, SDK for custom UX, API for server-side checks, MCP for AI/agent integrations.
  </Step>
  <Step title="Keep `/verify` as the canonical hosted path">
    Interactive flows should converge on the hosted verify surface instead of inventing parallel auth or checkout routes.
  </Step>
  <Step title="Use proof receipts as the single source of truth">
    Status, access, and policy should key off the same proof receipt instead of duplicating verification logic.
  </Step>
</Steps>

## Decision Matrix

| If you need... | Start here | Then add |
|---|---|---|
| Fastest production launch | [Widgets](./widgets/overview) | [SDK](./sdks/overview) for custom UX |
| Custom browser UX | [SDK](./sdks/overview) | [API](./api/overview) for server checks |
| Server-only eligibility checks | [API](./api/overview) | [Cookbook](./cookbook/overview) for rollout patterns |
| AI assistant or agent workflows | [MCP](./cookbook/mcp) | [Agents](./agents/overview) for proof models |
| Org/app onboarding and sponsored flows | [Partner Setup](./cookbook/hub-integrator) | [Auth + Hosted Verify](./cookbook/auth-hosted-verify) |
