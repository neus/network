Choose the fastest NEUS surface for your product, team, or agent workflow.

## Choose by Build Style

| Use case | Surface | Result |
|---|---|---|
| Most browser products | Hosted verify | Fastest path for guided login and interactive verification |
| React apps gating pages or routes | Widgets | Lowest-code path for proof-aware UI |
| Custom browser or server integration | SDK | Direct control over request creation and proof reuse |
| Server-only policy checks | API | Backend-owned trust decisions and policy checks |
| AI or agent tools | MCP | Verifier lookup, proof checks, and hosted handoff |
| Partner onboarding and domain setup | Partner Setup | Domain, team, app, and rollout setup |

## Decision Rules

1. Use hosted verify when people need to sign in, link accounts, or complete an interactive step.
2. Use widgets when the job is mostly gating or proof display inside a React app.
3. Use the SDK when you need a custom app experience but still want the standard NEUS contract.
4. Use the API when your server is making the trust decision.
5. Use MCP when an agent or editor tool needs the same trust layer.
