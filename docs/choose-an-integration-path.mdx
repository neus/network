**New integration?** Complete **[Get started](./get-started)** (platform profile, app link, credits) before you choose an integration path.

## Options

| You need | Use | Why |
|----------|-----|-----|
| Guided verification / login | [Hosted Verify](./cookbook/auth-hosted-verify) | Users finish in NEUS; you use `returnUrl` + session |
| React gating or badges | [Widgets](./widgets/overview) | `VerifyGate`, `ProofBadge` |
| Custom flow in code | [SDK](./sdks/overview) | Full control |
| Server-only | [API](./api/overview) | HTTP from your servers |
| AI / IDE tools | [MCP](./mcp/overview) | Add one URL; **open hosted links** tools return; optional Bearer like the SDK |

**MCP:** URL alone does not sign the IDE in — for “who am I” after login, same **`Authorization: Bearer`** as [SDK Authentication](./sdks/authentication).

**Partner / domain setup:** [Partner setup](./cookbook/hub-integrator).

**Who pays:** [Get started](./get-started) checklist + [Billing](./platform/billing) reference.
