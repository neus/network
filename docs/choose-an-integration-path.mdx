Not sure where to start? Pick the path that matches your product, then move to production with the same receipt model.

<Tip>
  **Shipping a product?** Start with **[Get started](./get-started)** first for app linking, credits, and billing.
</Tip>

## Pick your path

| You need | Use | Best for |
|----------|-----|----------|
| Guided verification and login | [Hosted Verify](./cookbook/auth-hosted-verify) | Fastest time-to-value |
| React gating or proof badges | [Widgets](./widgets/overview) | One component in your app |
| Custom app or server flow | [SDK](./sdks/overview) | Full control in code |
| Direct server-side checks | [API](./api/overview) | Backend and automation |
| Agent tools and hosted handoff | [MCP](./mcp/overview) | Cursor, Claude, and other clients |
| Credits, sponsor grants, and x402 | [Billing](./platform/billing) | Who pays and how requests settle |

## Decision rule

1. If you want the shortest path to a live flow, choose Hosted Verify.
2. If you are building a React app, choose Widgets.
3. If you need backend control, choose SDK or API.
4. If an agent will call NEUS tools, choose MCP.
5. If you are shipping a product, set up billing before volume.
