Create a proof, save the proof receipt ID, and reuse it in a product flow.

## 1. Install the SDK

```bash
npm install @neus/sdk
```

## 2. Create a Client

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({ appId: 'acme-web' });
```

## 3. Create a Proof

Use the ownership and provenance verifier for the first build. It is the simplest way to bind a wallet to content or a claim and get back a reusable proof receipt.

```javascript
const proof = await client.verify({
  verifier: 'ownership-basic',
  content: 'Hello NEUS',
  wallet: window.ethereum,
});

const proofId = proof.proofId;
```

<Note>
Public docs call this value the proof receipt ID. The SDK field name is `proofId`.
</Note>

## 4. Save the Proof Receipt ID

Persist it with the product record that will need trust later, such as a submission, profile, claim, or access decision.

## 5. Check Status Only When Needed

Some flows complete immediately. Others return `202` and finish asynchronously.

```javascript
const final = await client.pollProofStatus(proofId, {
  interval: 3000,
  timeout: 60000,
});

console.log(final.status);
```

## 6. Reuse It

Use the saved proof receipt ID to show a proof page, run a server check, or gate UI. For React gating, `VerifyGate` is the fastest path:

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

export default function ProtectedPage() {
  return (
    <VerifyGate
      requiredVerifiers={['nft-ownership']}
      verifierData={{
        'nft-ownership': {
          contractAddress: '0x...',
          tokenId: '1',
          chainId: 1,
        },
      }}
    >
      <div>Access granted.</div>
    </VerifyGate>
  );
}
```

## When To Use Hosted Verify

- Prefer hosted verify when the flow is interactive or user-guided.
- Save the proof receipt ID after completion.
- Reuse existing proofs before asking the user to verify again.
