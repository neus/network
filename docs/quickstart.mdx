**Default:** NEUS issues receipts. Passkey, wallet, and OAuth for standard browser flows run on **`https://neus.network/verify`**, not inside your application bundle.

| Outcome | You ship | Wallet or passkey code in your app |
|---------|----------|-------------------------------------|
| User proves something in the browser | `VerifyGate` **or** `getHostedCheckoutUrl` → `/verify`, then store **`qHash`** from callback or popup | No |
| Your backend creates a receipt for a user already linked in NEUS | `verifyFromApp`, then `gateCheck` | No |

## Browser

| Your responsibility | NEUS |
|--------------------|------|
| `appId`, `returnUrl` (redirect) **or** `origin` + popup listener | Ceremony UI and signing |

**VerifyGate** (React):

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

<VerifyGate appId="your-app-id" requiredVerifiers={['ownership-basic']}>
  <ProtectedContent />
</VerifyGate>
```

**Hosted URL** (any framework):

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

window.location.assign(
  getHostedCheckoutUrl({
    verifiers: ['ownership-basic'],
    returnUrl: 'https://myapp.com/auth/callback',
  }),
);
```

Parameters, `returnTo`, `intent=login`, popups (`neus_checkout_done` / `neus_login_complete`): [Hosted Verify](./cookbook/auth-hosted-verify).

## Server

```env
NEUS_APP_ID=app_...
NEUS_ACCESS_KEY=npk_...
```

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: process.env.NEUS_APP_ID,
  apiKey: process.env.NEUS_ACCESS_KEY,
});

await client.verifyFromApp({
  user: { walletAddress: user.walletAddress },
  verifier: 'ownership-basic',
  content: { claim: 'User completed onboarding', source: 'your-app' },
});

const result = await client.gateCheck({
  address: user.walletAddress,
  verifierIds: ['ownership-basic'],
});
```

## Assistants (MCP)

Connect editors and agents to hosted NEUS MCP with one command (then start each session with tool **`neus_context`**):

```bash
npx -y -p @neus/sdk neus setup
```

Optional Profile access key (for tools such as `neus_me`):

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

Health check:

```bash
npx -y -p @neus/sdk neus doctor
```

Install, JSON, headers, and per-client paths: [MCP setup](./mcp/setup) · Discovery URLs: [MCP endpoints](./mcp/endpoints) · Session tool order: [MCP overview](./mcp/overview).

**Delegated agent wallet (identity + access proofs):** follow the [Agent verification flow](./agents/agent-verification-flow). Reuse **`neus_agent_link`** before **`neus_agent_create`** when a wallet already exists.
