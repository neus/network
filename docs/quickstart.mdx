NEUS works with the users you already have. Your app keeps its login; NEUS adds reusable trust receipts when you need proof, access, or trust checks.

Connect your app once, then use your server to create and check trust receipts. Users do not need to sign every request.

## 1. Connect your app

Create an **App Link** from your NEUS profile. This gives your server an app ID, access key, and App Link receipt.

Store them as server-side environment variables:

```env
NEUS_APP_ID=app_...
NEUS_ACCESS_KEY=npk_...
```

## 2. Verify from your server

When a user is already signed into your app, your server can ask NEUS to create a trust receipt.

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: process.env.NEUS_APP_ID,
  apiKey: process.env.NEUS_ACCESS_KEY,
});

const receipt = await client.verifyFromApp({
  user: { walletAddress: user.walletAddress },
  verifier: 'ownership-basic',
  content: {
    claim: 'User completed onboarding',
    source: 'your-app',
  },
});
```

`verifyFromApp` sends your app ID with the request. NEUS checks that this user linked your app, then creates the receipt. No wallet popup. No extra NEUS login UI inside your app.


## 3. Check a gate

Later, check whether the user has the required receipt:

```javascript
const result = await client.gateCheck({
  address: user.walletAddress,
  verifierIds: ['ownership-basic'],
});

if (!result.data?.eligible) {
  throw new Error('Trust receipt required');
}
```

## Hosted Verify (no wallet in your UI)

For browser flows where NEUS should handle login or provider steps, send users to Hosted Verify. The hosted path is passkey-first, with wallet connection only when a verifier needs it.

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

const url = getHostedCheckoutUrl({
  verifiers: ['ownership-basic'],
  returnUrl: 'https://myapp.com/neus/callback',
});

window.location.assign(url);
```

Save the returned `qHash` in your database keyed to your own `userId`. It is the portable trust receipt reference.

[Hosted Verify](./cookbook/auth-hosted-verify) covers redirect and popup return handling.

### React: gate in place

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

<VerifyGate
  appId="your-app-id"
  requiredVerifiers={['ownership-basic']}
>
  <ProtectedContent />
</VerifyGate>
```

## Direct wallet signing

When you need the user to sign with a browser wallet inside your app:

```javascript
const client = new NeusClient({ appId: 'your-app-id' });

const proof = await client.verify({
  verifier: 'ownership-basic',
  content: 'My content or claim',
  wallet: window.ethereum,
});

const qHash = proof.qHash;
```

`client.verify()` defaults **private**. See [SDK authentication](./sdks/authentication) and [Signing format](./verification/signing-format).

## Polling (optional)

If you need to wait on async work:

```javascript
const status = await client.pollProofStatus(qHash, {
  interval: 3000,
  timeout: 60000,
});
```
