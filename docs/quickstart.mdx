This page is for your **first direct-sign proof** with **`ownership-basic`**. Flows that need social, human, or org interaction start in **[Hosted Verify](./cookbook/auth-hosted-verify)**; reusable gate flows follow **[Integration](./integration)**.

Before production traffic, complete **[Get started](./get-started)** for `appId`, credits, and who pays.

## 1. Install the SDK

```bash
npm install @neus/sdk
```

## 2. Create a proof

**Raw `client.verify()`** defaults to **private**, **unlisted**, and **stored content** (owner-authenticated reads). Omit `options` unless you need a different mode. **`VerifyGate`** uses different create defaults; see **[Widgets](./widgets/overview)** and **[Security and trust](./platform/security-and-trust)**.

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: 'your-app-id', // optional for local try; required for production — [Get started](./get-started)
});

const proof = await client.verify({
  verifier: 'ownership-basic',
  content: 'My content or claim',
  wallet: window.ethereum,
});

const proofId = proof.proofId;
```

**Save `proofId` on your server with the same user you verified.** It is the **proof receipt ID** (many API payloads also use **`qHash`** for the same value). You will reuse it for checks and receipts—not the raw signature string.

**Attestation vs gate:** this snippet is for **proving content**. See **[SDK verification patterns](./sdks/verifications)** for more shapes.

No injected wallet? Use **[Hosted Verify](./cookbook/auth-hosted-verify)** instead of `window.ethereum`.

Social, org, or human verification? Start in **[Hosted Verify](./cookbook/auth-hosted-verify)**, not here.

## 3. Wait for completion

```javascript
const status = await client.pollProofStatus(proofId, {
  interval: 3000,
  timeout: 60000,
});
```

## 4. Gate content in React

For gates, **`VerifyGate`** defaults to **unlisted public** create mode so checks align without extra options. Override **`proofOptions`** when you need listed public or private behavior. See **[VerifyGate](./widgets/verifygate)** and **[Integration](./integration)**.

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

<VerifyGate
  appId="your-app-id"
  requiredVerifiers={['ownership-basic']}
>
  <ProtectedContent />
</VerifyGate>
```

## 5. Check eligibility from your server

**Rules that prevent “it doesn’t work” bugs:**

- **`gateCheck` `address` must be the same identity** that signed **`verify()`** (for `ownership-basic`, the wallet you passed as `wallet`).
- **Prefer a stored proof receipt ID:** call the proofs check API (via **`gateCheck`** or **`GET /api/v1/proofs/check`**) **before** you show another verify flow when you already created a proof for this user.
- **Verifier policy must match:** same `verifierIds` (and equivalent `verifierData` constraints) you used at creation time.

```javascript
const signerAddress = '0x...';

const result = await client.gateCheck({
  address: signerAddress,
  verifierIds: ['ownership-basic'],
});

if (result.data?.eligible) {
}
```
