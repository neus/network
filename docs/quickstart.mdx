For hosted login and verify, use **[Hosted Verify](./cookbook/auth-hosted-verify)** or **[MCP](./mcp/overview)**. Before production traffic, complete **[Get started](./get-started)** (**`appId`**, credits, who pays).

## 1. Install the SDK

```bash
npm install @neus/sdk
```

## 2. Create a proof

Defaults are **private**, **unlisted**, and **original content stored** (owner-authenticated reads). Omit `options` unless you need a different mode.

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: 'your-app-id', // optional for local try; required for production — [Get started](./get-started)
});

const proof = await client.verify({
  verifier: 'ownership-basic',
  content: 'My content or claim',
  wallet: window.ethereum,
});

const proofId = proof.proofId;
```

**Attestation vs gate:** this snippet is for **proving content**. For **access gates** that must **`gateCheck`** without the owner in-session, use **public unlisted** (`privacyLevel: 'public'`, `publicDisplay: false`)—see step 4 and **[SDK verification patterns](./sdks/verifications)**.

No injected wallet? Use **[Hosted Verify](./cookbook/auth-hosted-verify)** instead of `window.ethereum`.

## 3. Wait for completion

```javascript
const status = await client.pollProofStatus(proofId, {
  interval: 3000,
  timeout: 60000,
});
```

## 4. Gate content in React

For **gates**, set **`proofOptions`** so eligibility checks line up (same pattern as **[Widgets](./widgets/overview)**).

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

<VerifyGate
  appId="your-app-id"
  requiredVerifiers={['ownership-basic']}
  proofOptions={{
    privacyLevel: 'public',
    publicDisplay: false,
  }}
>
  <ProtectedContent />
</VerifyGate>
```

More props: **[VerifyGate](./widgets/verifygate)**.

## 5. Check eligibility from your server

```javascript
const result = await client.gateCheck({
  address: '0x...',
  verifierIds: ['ownership-basic'],
});

if (result.data?.eligible) {
  // allow access
}
```
