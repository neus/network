`npm install` → one `verify()` → read `proofId` → poll if needed → gate or `gateCheck`. ~5 minutes.

<Tip>
  Snippets use **`new NeusClient()`** without **`appId`**—fine for a first proof. For production, register an app and pass **`appId`** — **[Get started](./get-started)**.
</Tip>

## 1. Install the SDK

```bash
npm install @neus/sdk
```

## 2. Create a proof

Defaults are **private**, **unlisted**, and **original content stored** (owner-authenticated reads). Omit `options` unless you need a different mode.

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient();

const proof = await client.verify({
  verifier: 'ownership-basic',
  content: 'My content or claim',
  wallet: window.ethereum,
});

const proofId = proof.proofId;
```

**Reusable gates** (same receipt for `gateCheck` without the owner signed in) use **public unlisted**: `privacyLevel: 'public'`, `publicDisplay: false`. Unlisted is still public to anyone with the proof id—use for non-sensitive gates. See **[SDK verification patterns](./sdks/verifications)**.

<Tip>
  **No in-app wallet?** Use **[Hosted Verify](./cookbook/auth-hosted-verify)** (redirect or popup) instead of `window.ethereum`.
</Tip>

## 3. Wait for completion

```javascript
const status = await client.pollProofStatus(proofId, {
  interval: 3000,
  timeout: 60000,
});
```

## 4. Gate content in React

`VerifyGate` uses the same defaults as the SDK unless you override `proofOptions`.

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

<VerifyGate
  requiredVerifiers={['ownership-basic']}
>
  <ProtectedContent />
</VerifyGate>
```

For a public unlisted gate, set `proofOptions` explicitly (example in **[VerifyGate](./widgets/verifygate)**).

## 5. Check eligibility from your server

```javascript
const result = await client.gateCheck({
  address: '0x...',
  verifierIds: ['ownership-basic'],
});

if (result.data?.eligible) {
  // allow access
}
```
