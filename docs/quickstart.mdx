NEUS runs sign-in and verification in the browser. Your app saves the receipt and checks it before payouts, access, or other trusted actions.

**End users never need a NEUS account, an API key, or extra setup in your product.**

## 1. Register your app

1. Sign in at [neus.network/hub](https://neus.network/hub).
2. Open **Apps** → **Register app**.
3. Enter your site URL and confirm once.
4. Copy your **app ID** (`appId`) and **billing wallet** (`billingWallet` — your Hub account address).

Checks that run through your product bill this account. Add credits under **Usage** in Hub.

## 2. Gate your UI

```jsx
import { VerifyGate } from "@neus/sdk/widgets";

<VerifyGate
  appId="your-app-id"
  billingWallet="0xYourHubWallet"
  requiredVerifiers={["ownership-basic"]}
>
  <ProtectedContent />
</VerifyGate>;
```

`VerifyGate` looks for an existing receipt, opens **Hosted Verify** when a new one is needed, then shows your content. Wallet, passkey, and OAuth all happen on NEUS — not inside your app.

## 3. Or send users to Hosted Verify

```javascript
import { getHostedCheckoutUrl } from "@neus/sdk";

window.location.assign(
  getHostedCheckoutUrl({
    verifiers: ["ownership-basic"],
    returnUrl: "https://myapp.com/auth/callback",
    appId: "your-app-id",
    billingWallet: "0xYourHubWallet",
  }),
);
```

Read **`qHash`** from the callback URL or popup message. Details: [Hosted Verify](./cookbook/auth-hosted-verify).

## 4. Check before trusted actions

On your server, confirm the receipt still satisfies your policy before you act:

```javascript
import { NeusClient } from "@neus/sdk";

const client = new NeusClient({
  appId: process.env.NEUS_APP_ID,
  billingWallet: process.env.NEUS_BILLING_WALLET,
});

const result = await client.gateCheck({
  address: user.walletAddress,
  verifierIds: ["ownership-basic"],
});

if (!result.data?.eligible) {
  // send the user back to VerifyGate or hosted verify
}
```

Use the same `appId` and `billingWallet` on the server so eligibility checks stay tied to your billing account.

## Assistants and agents

Connect MCP hosts:

```bash
npx -y -p @neus/sdk neus setup
npx -y -p @neus/sdk neus auth
npx -y -p @neus/sdk neus doctor --live
```

Use **`neus auth`** for OAuth in your browser. Access keys are for servers, CI, and automation — never in browser code. Call **`neus_context`** first in each MCP session.

Agent wallets: [Agent verification flow](./agents/agent-verification-flow).
