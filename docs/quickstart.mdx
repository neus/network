This guide is the shortest path from zero setup to a reusable NEUS proof in a real app flow.

## What You'll Do

1. Install the SDK
2. Create a proof with `ownership-basic`
3. Save the returned proof receipt ID
4. Poll status for async flows
5. Gate UI or server access with that receipt

## 1. Install the SDK

```bash
npm install @neus/sdk
```

## 2. Create a Client

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({ appId: 'neus-network' });
```

## 3. Create a Proof

Create a basic ownership proof. This prompts the user to sign a request-bound message.

```javascript
const proof = await client.verify({
  verifier: 'ownership-basic',
  content: 'Hello NEUS',
  wallet: window.ethereum,
});

const proofId = proof.proofId;
```

<Note>
`proofId` is the standard SDK name for the proof receipt ID. Raw HTTP paths may use `{qHash}` for the same identifier.
</Note>

## 4. Poll Status

Some verifiers complete immediately. Others return `202` and finish asynchronously.

```javascript
const final = await client.pollProofStatus(proofId, {
  interval: 3000,
  timeout: 60000,
});

console.log(final.status);
```

## 5. Gate Access

Use `VerifyGate` when you want the UI to check eligibility before showing content.

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

export default function ProtectedPage() {
  return (
    <VerifyGate
      requiredVerifiers={['nft-ownership']}
      verifierData={{
        'nft-ownership': {
          contractAddress: '0x...',
          tokenId: '1',
          chainId: 1,
        },
      }}
    >
      <div>Access granted.</div>
    </VerifyGate>
  );
}
```

## Hosted Interactive Verifiers

For `ownership-social`, `ownership-org-oauth`, and `proof-of-human`, use the hosted NEUS flow.

```jsx
<VerifyGate
  requiredVerifiers={['ownership-social']}
  onVerified={(result) => {
    console.log('Hosted verification complete', result.proofId);
  }}
>
  <div>Access granted.</div>
</VerifyGate>
```

## Production Defaults

- Prefer private proofs unless you explicitly need public discovery.
- Persist the proof receipt ID, not raw signature material.
- Use freshness windows for point-in-time verifiers such as balances, ownership, and risk.
