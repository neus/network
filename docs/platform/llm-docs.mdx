## MCP tool order

1. `https://mcp.neus.network/mcp`. [Setup](../mcp/setup)
2. **`neus_context`**
3. Bearer, then **`neus_me`** (`status: ok`). [Auth](../mcp/auth)
4. **`neus_proofs_check`**, **`neus_verify`**, **`neus_verify_or_guide`**, agents. [Overview](../mcp/overview)
5. **`neus_proofs_get`**. [Proofs get](../mcp/proofs-get)

`npx -y -p @neus/sdk neus init` installs the hosted NEUS MCP server into supported clients automatically. Use `neus auth --access-key <npk_...>` when you want account-scoped tools such as `neus_me` and private reads, then `neus status --json` to confirm the final client state or surface any malformed local config.

## Rules

- **Browser → NEUS:** SDK (`NeusClient`, widgets) or **Hosted Verify**. Do not call `api.neus.network` from the browser with secrets or ad-hoc headers; use a **server proxy** when you need custom server-side calls.
- **Server → NEUS:** `gateCheck`, eligibility, secrets, raw HTTP — [API overview](../api/overview)
- Account, `appId`, and billing: [Get started](../get-started)
- Default browser ceremony: [Hosted Verify](../cookbook/auth-hosted-verify). Optional in-app signing: [Signing format](../verification/signing-format)
- Do not use wallet **private keys** as MCP credentials; use **personal access keys** (`npk_*`) for Bearer. Agent signing keys stay in your runtime ([Agent create](../mcp/agent-create) may return a generated key for integrators only).
- **402:** use the x402 retry pattern
- Agents: [Overview](../agents/overview) · [Verification flow](../agents/agent-verification-flow)
- Reuse **`qHash`**

## MCP JSON

**`type`** is client-specific; use **`http`** for URL-based hosted MCP.

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp"
    }
  }
}
```
