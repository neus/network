NEUS publishes route indexes and full-doc exports for tools that ingest documentation text.

## Reader order (integrators)

1. **[Overview](../index)** — proof receipt ID model (`proofId` / `qHash`).
2. **[Get started](../get-started)** — hub account, **`appId`**, credits, sponsor vs self-pay.
3. **[Integration](../integration)** — check → verify → save → reuse.
4. Path docs (SDK, API, Hosted Verify, MCP, widgets) — only after **2** is clear.

**Assistants:** If **`appId`**, credits, billing, or **`sponsorGrant`** is ambiguous, ground answers in **[Get started](../get-started)** before linking to MCP tools or API details.

## Route index (`llms.txt`)

```bash
curl https://docs.neus.network/llms.txt
```

## Full documentation (`llms-full.txt`)

```bash
curl https://docs.neus.network/llms-full.txt
```

## MCP

Add the hosted MCP server for live verification tools:

```json
{
  "mcpServers": {
    "neus": {
      "type": "streamableHttp",
      "url": "https://mcp.neus.network/mcp"
    }
  }
}
```
