## Start from the user outcome

1. Define what must be true for access, trust, or compliance.
2. Select the verifier that proves that outcome.
3. Save the returned proof receipt ID and reuse it in checks, UI, and policy decisions.

## Common goals

| Goal | Typical verifier(s) |
|------|----------------------|
| Confirm a real person | `proof-of-human` |
| Check token or NFT access | `token-holding`, `nft-ownership` |
| Verify ownership of content or identity | `ownership-basic`, `ownership-social`, `ownership-org-oauth` |
| Confirm domain ownership | `ownership-dns-txt` |
| Register and delegate agents | `agent-identity`, `agent-delegation` |

## Selection tips

- Prefer the smallest verifier set that satisfies your policy.
- Use hosted verification for interactive steps such as social or organization checks.
- Use fresh proofs only for high-stakes operations; reuse existing receipts for routine checks.
