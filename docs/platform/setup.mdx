## DNS domain

TXT at `_neus.<domain>`:

```text
neus=<your-wallet-address>
```

Then prove (example):

```javascript
const proof = await client.verify({
  verifier: 'ownership-dns-txt',
  data: { domain: 'yourdomain.com', walletAddress: '0x...' },
  walletAddress: '0x...',
  signature: '0x...',
  signedTimestamp: Date.now(),
});
```

Confirm proofs the same way your app will before gating - e.g. [Dashboard](https://neus.network/hub) or the check step in [Integration](../integration).

## Organization

Google Workspace / Microsoft 365 to hosted verify; NEUS runs OAuth.

1. `https://neus.network/verify`
2. `preset=org-pack` when membership alone is enough
3. `returnUrl` or popup
4. Store **verification ID**

```text
https://neus.network/verify?preset=org-pack&returnUrl=https://app.example.com/verified
```

```text
https://neus.network/verify?preset=org-pack&mode=popup&origin=https://app.example.com
```

One company only:

```jsx
<VerifyGate
  hostedCheckoutUrl="https://neus.network/verify"
  requiredVerifiers={['ownership-org-oauth']}
  verifierData={{
    'ownership-org-oauth': { provider: 'google', expectedOrgDomain: 'acme.com' },
  }}
>
  <ProtectedContent />
</VerifyGate>
```

You rarely need your own Google/Microsoft OAuth app.

## Attribution

`appId` is public - not secret. The SDK attaches it on your behalf when you construct the client with your id.

```javascript
const client = new NeusClient({ appId: 'your-app-id' });
```

## Sponsor grants

```javascript
new NeusClient({ appId: 'your-app-id', sponsorGrant: '<token>' });
```

<Warning>
Short-lived tokens. Re-issue per session. No long-lived secrets in client bundles.
</Warning>
