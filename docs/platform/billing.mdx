Credits power verification and gate checks. A request can be paid by the user, by a sponsoring app, or through x402 pay-per-use.

NEUS uses one billing model across verification and gating.

## Payment Options

| Option | Best for |
|---|---|
| Sponsor grants | Your app covers the cost for users |
| User credits | The user pays from their own balance |
| x402 pay-per-use | Pay at request time when credits are not available |

## Sponsor Grants

Apps can sponsor verifications for their users.

- **Header:** `X-Sponsor-Grant: <jwt>`
- **Use case:** Onboarding where the app covers verification cost

## User Credits

Users can pay from their own NEUS credit balance when the integration allows self-pay.

## x402 Pay-Per-Use

When credits are exhausted, the API returns `402 Payment Required` with a quote. The client can pay via crypto and retry with the receipt.

- **Header:** `PAYMENT-SIGNATURE: <receipt>`
- **Use case:** One-off usage without pre-loading credits

## Public vs Private

- **Public proofs:** Anyone can check via `/api/v1/proofs/check` (no auth required)
- **Private proofs:** Owner-only access; requires authenticated session or owner signature

Billing and access control are separate concerns.
