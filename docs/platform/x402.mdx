NEUS exposes a standards-native x402 v2 resource for agents and server applications that need a trust decision before taking an action. A caller can discover the price, authorize USDC, retry the same request, and receive the result without creating a NEUS account.

## Payable HTTP resource

The canonical x402 resource is:

```text
GET https://api.neus.network/api/v1/proofs/check
```

Pass either a deployed `gateId` or explicit receipt criteria. This example checks whether an account has a reusable `ownership-basic` receipt:

```bash
curl -i --get https://api.neus.network/api/v1/proofs/check \
  --data-urlencode "address=0xYOUR_ACCOUNT" \
  --data-urlencode "verifierId=ownership-basic"
```

<Note>
  `https://mcp.neus.network/mcp` is the authenticated MCP transport for MCP clients. HTTP x402 clients should use the `api.neus.network` resource above.
</Note>

## Standards-native v2 flow

<Steps>
  <Step title="Request the resource">
    Send the final HTTP method, URL, query parameters, and body you intend to use.
  </Step>
  <Step title="Read the payment requirements">
    An unpaid request returns HTTP `402`. The `PAYMENT-REQUIRED` response header contains a base64-encoded x402 v2 `PaymentRequired` object with `x402Version: 2`, the resource, and an `accepts` array.
  </Step>
  <Step title="Authorize payment">
    Select the advertised `exact` requirement and let an x402 v2 client create the payment payload. NEUS currently accepts USDC settlement on Base for production.
  </Step>
  <Step title="Retry the identical request">
    Repeat the same request with the base64-encoded payment payload in `PAYMENT-SIGNATURE`. NEUS verifies it before execution and settles it after a successful call.
  </Step>
  <Step title="Keep the settlement receipt">
    A successful paid response includes `PAYMENT-RESPONSE`, a base64-encoded settlement result containing the network and transaction reference.
  </Step>
</Steps>

The three x402 v2 transport headers are:

| Header | Direction | Purpose |
| --- | --- | --- |
| `PAYMENT-REQUIRED` | NEUS → caller | Base64-encoded price and payment requirements on HTTP 402 |
| `PAYMENT-SIGNATURE` | Caller → NEUS | Base64-encoded payment payload on the retry |
| `PAYMENT-RESPONSE` | NEUS → caller | Base64-encoded settlement result |

Do not parse these headers as raw JSON. Use an x402 v2 client or the header helpers from `@x402/core/http`.

## Bazaar discovery

The trust-check resource declares the official x402 v2 Bazaar extension with:

- a semantic description oriented around pre-action trust and authority decisions;
- a realistic query example;
- JSON Schema for supported input fields;
- output metadata, schema, and an example result;
- an `exact` USDC payment requirement on a CAIP-2 Base network.

Compatible facilitators can index the declared resource after a paid call settles. Discovery catalogs refresh asynchronously, so a newly settled resource may not appear immediately.

## Agents that pay on their own

Pair x402 with an `agent-delegation` receipt to constrain autonomous spend:

- set `scope: "payments:x402"`;
- restrict `allowedPaymentTypes` to `x402`;
- store `maxSpend` as a whole-number string in token base units (USDC has 6 decimals);
- reject payment when the requested amount exceeds the remaining delegated cap.

Use `toAgentDelegationMaxSpend('25', 6)` from `@neus/sdk` to represent a 25 USDC cap as `"25000000"`.
