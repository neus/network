**Default path:** Platform credits + **sponsor grants** — **[Get started](../get-started)**. **x402** is for when you are **out of credits** and need to **pay per request**, then retry with a receipt.

x402 lets a request pay at the time it runs. If credits are not available, the API returns a quote, the client pays, and the request is retried with the receipt.

## Where x402 Applies

| API or flow | When 402 happens |
|---------|------------------|
| **Verification API** | `POST /api/v1/verification` when the caller has no credits |
| **Gate check API** | `GET /api/v1/proofs/check` when the caller has no credits |
| **MCP tools** | Tool call returns payment-needed when no credits |
| **Hosted flows** | When the calling app or server makes billable requests without credits |

## Flow

1. **Request** — Call verification or gate check.
2. **402** — API returns `402 Payment Required` with quote.
3. **Pay** — Pay the quoted amount.
4. **Retry** — Resend request with `PAYMENT-SIGNATURE: <receipt>`.

## Using x402 Through Verifiers and Checkout

- **Verifiers** — Direct API verification can fall back to x402 when credits are missing.
- **Gates** — Gate checks can do the same.
- **Hosted flows** — x402 matters when the calling app or server is the billable party.
- **Agents** — Delegation can allow pay-per-use on behalf of the approving account.

## Header

```text
PAYMENT-SIGNATURE: <receipt>
```
