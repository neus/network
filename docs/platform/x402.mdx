title: x402 Pay-Per-Use
description: Retry with payment when credits run out.
icon: coins
---

Default: org **credits** and **`sponsorGrant`** when you bill customer traffic ([Get started](../get-started), [Billing](./billing)).

**x402:** API returns **402** and a quote; retry with **`PAYMENT-SIGNATURE`**. Same account model as the rest of billing, not a separate product.

## When

| | |
|--|--|
| `POST /api/v1/verification` | No credits |
| `GET /api/v1/proofs/check` | No credits |
| MCP | No credits |
| Other billable calls | No credits on payer |

## Flow

1. Call API 
2. **402** + quote 
3. Pay 
4. Retry with `PAYMENT-SIGNATURE: <receipt>`

Same idea for verification, checks, hosted-adjacent calls, and agent flows that allow x402.

## Header

```text
PAYMENT-SIGNATURE: <receipt>
```
