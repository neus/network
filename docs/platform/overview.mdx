## Core Product Loop

1. **Verify** through the hosted NEUS flow or SDK
2. **Receive** a proof receipt ID
3. **Share or display** the proof page and badge
4. **Reuse** the same proof receipt for access, monetization, and policy

## Why This Matters

| Need | NEUS surface |
|------|--------------|
| Frictionless onboarding | Hosted verify and login |
| Public trust page | Standard proof page |
| Identity and app management | Hub and profile surfaces |
| Reusable business logic | Proof receipt reuse across products |
