## Boundaries

- Proof issuance runs in **NEUS**.
- Return **only** what your product needs from checks.
- **Private** reads need the owner signed in (or your sharing rules).

## Modes

| Mode | `privacyLevel` | `publicDisplay` | Use |
|------|----------------|-----------------|-----|
| Listed public | `public` | `true` | Discoverable |
| Unlisted public | `public` | `false` | Gates, reuse, not listed |
| Owner-only | `private` | `false` | Owner auth |

Saved **verification IDs** are capability-bearing links for the mode you chose.

`privacyLevel`: vaulted vs checkable without sign-in. `publicDisplay`: listed vs not. Unlisted public still works for anyone with the id.

## Defaults

| Surface | Create default |
|---------|----------------|
| `client.verify()` | `private`, `storeOriginalContent: true` |
| VerifyGate | unlisted public (`public` + `publicDisplay: false` + `storeOriginalContent: true`) |

Override with `proofOptions` when you need listed public or stricter private.

## storeOriginalContent

| | Effect |
|--|--------|
| `true` (default) | Keep content context for verification. |
| `false` | Hash-first where your verifier allows - know the tradeoff. |

Wallet-signed **`ownership-basic`** (API / MCP): storage and listed-public behavior. [Ownership and Provenance](../verification/ownership-basic).

## Never ship publicly

Raw signatures, owner-only payloads, private proof enumeration.
