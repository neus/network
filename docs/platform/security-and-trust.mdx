Use this page when you pick public, unlisted, or private behavior. Shorter guides link here for the full matrix and defaults.

## Trust boundaries

- Verification completes in **NEUS**. Do not treat the browser alone as proof of issuance.
- Gate checks and status should return **only** what your product needs.
- **Private** proof reads require the **owner signed in** (or your product’s explicit sharing rules).

## Visibility modes

| Mode | `privacyLevel` | `publicDisplay` | When to use |
|------|----------------|-----------------|-------------|
| Listed public | `public` | `true` | Publishing and attribution where the proof should be discoverable |
| Unlisted public | `public` | `false` | Non-sensitive reuse, gates, and checks without listing for discovery |
| Owner-only | `private` | `false` | Owner-only access and owner-authenticated reuse |

Anyone with the proof receipt ID can fetch metadata appropriate to that mode. Treat IDs like capability-bearing links.

**Fields:** `privacyLevel` chooses vaulted (private) versus checkable without a signed-in owner (public). `publicDisplay` chooses discoverable listing. Unlisted public proofs are still public to anyone who has the receipt id.

## Default behavior by surface

| Surface | Create default | Notes |
|---------|----------------|-------|
| **Raw SDK** `client.verify()` | `privacyLevel: 'private'`, `publicDisplay: false`, `storeOriginalContent: true` | Private-by-default for server and wallet flows |
| **`VerifyGate`** create mode | `privacyLevel: 'public'`, `publicDisplay: false`, `storeOriginalContent: true` | Unlisted public for reuse-first gating |

Override with explicit `proofOptions` / `options` when your product needs listed public or stricter private behavior.

## storeOriginalContent

| Value | Effect |
|-------|--------|
| `true` | Default. Retains original content for verification context unless you opt out. |
| `false` | Hash-only or minimal retention where your flow allows it. |

Use `false` only when you intentionally want hash-first storage and accept the tradeoffs for your verifier and policy.

## What should never be exposed publicly

- Raw signatures
- Owner-only proof payloads
- Private proof enumeration
