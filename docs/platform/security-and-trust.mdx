## Trust boundaries

- Verification completes in **NEUS** — do not treat the browser alone as proof of issuance.
- Gate checks and status should return **only** what your product needs.
- **Private** proof reads require the **owner signed in** (or your product’s explicit sharing rules).

## Privacy Defaults

Use **unlisted** proofs by default for reusable integrations. Use **private** when only the **signed-in user** should see the proof.

| Setting | Effect |
|---|---|
| `privacyLevel: 'private'` | Visible when that user is signed in with NEUS |
| `publicDisplay: false` | Keeps a public proof usable without making it discoverable |
| `storeOriginalContent: true` | Keeps original content when your product needs it |

## What Should Never Be Exposed Publicly

- Raw signatures
- Owner-only proof payloads
- Private proof enumeration

## Production Checklist

<Checklist>
  <Item>Use gate checks for access decisions.</Item>
  <Item>Respect time windows for point-in-time verifiers.</Item>
  <Item>Route interactive verification through `/verify`.</Item>
  <Item>Consume verifier metadata from the API catalog.</Item>
  <Item>Default to unlisted proofs unless public discovery is intentional.</Item>
</Checklist>
