A NEUS profile can manage **multiple agents** under one approving account. Each agent gets its own identity and permission receipt. Agents may share an operating wallet while keeping different limits.

## The model

One account approves multiple agents. Each agent has a separate permission receipt with its own `allowedActions`, `deniedActions`, and `scope`.

```text
Approving account
  ├── agent-a   → reporting permissions
  ├── agent-b   → content permissions
  └── agent-c   → operations permissions
```

The permission receipt, not the shared key, separates the agents. Each integration checks the receipt before allowing an action.

## Why one wallet can support several agents

- **No key sprawl.** One wallet to manage, secure, and rotate.
- **Per-agent limits.** Give each agent only the actions, spending cap, and time window it needs.
- **Traceable approvals.** Each permission receipt identifies the agent, approving account, and limits.
- **Focused revocation.** Revoke one agent&apos;s permissions without changing the others.

## Set it up

<Steps>
  <Step title="Create each agent">
    Call **`neus_agent_create`** for each team agent. Pass `agentId`, `scope`, `delegationAllowedActions`, `delegationDeniedActions`, `skills`, and `instructions`. The approving account signs the permission receipt.
  </Step>
  <Step title="Confirm each agent">
    Call **`neus_agent_link`** for each agent until **`linked: true`**.
  </Step>
  <Step title="Connect each project">
    In each project where the agent should operate:
    ```bash
    neus mount <agentId> --apply cursor
    ```
    Loads the agent&apos;s identity, permissions, skills, and context into the project.
  </Step>
  <Step title="Assign jobs">
    Add recurring jobs to the agent schedule. Each job includes an `agentId` so it runs under the correct permissions.
  </Step>
</Steps>

## Skills

Each agent can list role-specific skills in its permission receipt. Skills describe capabilities; `allowedActions` and `deniedActions` determine what the agent may actually do.
