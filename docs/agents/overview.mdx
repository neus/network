Name the agent. Set what it may do and how much it may spend. Connected runtimes enforce that authority before access or a sensitive tool runs.

| Need | How |
|------|-----|
| Identify the agent | Agent signs its identity |
| Set what it may do | Approving account grants authority |
| Cap spend | `maxSpend` in base units |
| Ready to act? | `neus_agent_link` → `linked: true` |
| Reuse later | Receipt ID or receipt URL |
| Audit | Receipt reads and proof pages |

Frameworks run tools. NEUS sets who the agent is and what it may do first.

## Two results make an agent ready

- **Identity:** the agent account signs its name, type, and skills. Identity alone grants no permission.
- **Authority:** an approving account signs allowed actions, spend cap, and expiration.

An agent is **ready** only when both are on file.

## Authority

The authority record defines a boundary:

- **`allowedActions` / `deniedActions`**: explicit allow and deny lists. A denied action takes precedence.
- **`maxSpend`**: a whole-number spending cap in token base units. USDC uses six decimals, so 25 USDC is `"25000000"`.
- **`scope`**: the area where permissions apply, such as `payments:x402` or `global`.
- **`expiresAt`**: expiration time in Unix milliseconds. Set it for payments or other sensitive actions.

Quick setup presets (`full-access`, `payments`, `readonly`, `automation`) are available through `neus_agent_create`. Full field reference: [Agent delegation](./agent-delegation).

## Set it up

<Steps>
  <Step title="Connect">
    ```bash
    neus setup
    neus check
    ```
    Configures hosted NEUS MCP for Cursor, Codex, VS Code, or Claude Code.
  </Step>
  <Step title="Check">
    Call **`neus_context`**, then **`neus_agent_link`** with the agent wallet.
  </Step>
  <Step title="Create">
    If identity or authority is missing, call **`neus_agent_create`**. The agent account signs its identity, and the approving account grants authority.
  </Step>
  <Step title="Confirm">
    Call **`neus_agent_link`** until **`linked: true`**, then reuse receipts.
  </Step>
  <Step title="Connect">
    In your project repo:
    ```bash
    neus mount <agentId> --apply cursor
    ```
    Loads the agent&apos;s verified identity, permissions, and rules into the project. [Connect agent context](./runtime-mount).
  </Step>
  <Step title="Act">
    Check receipt status before payments, data access, or other sensitive actions.
  </Step>
</Steps>
