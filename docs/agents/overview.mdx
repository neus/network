Verifiable agents with scoped permissions and receipts you can audit later.

## How agents work

1. Add the agent identity for the agent wallet.
2. Approve what it can do from the controller wallet.
3. Use Hosted Verify for interactive steps when a human needs to approve the action.

Different people can do steps 1 and 2. See [Agent concepts](./concepts) for roles, wallets, and API field names.

## Why agent verification matters

| Question | NEUS answer |
|----------|-------------|
| Which agent is acting? | Agent registration binds the agent to a wallet |
| What can it do? | Delegation scopes permissions |
| What happened later? | Proof receipts and MCP tools preserve the audit trail |

## Quick start

### Register an agent

```javascript
const agent = await client.verify({
  verifier: 'agent-identity',
  data: {
    agentId: 'my-agent',
    agentWallet: agentWallet,
    agentLabel: 'My AI Agent',
    agentType: 'ai',
  },
  walletAddress: agentWallet,
});
```

### Delegate permissions

```javascript
const delegation = await client.verify({
  verifier: 'agent-delegation',
  data: {
    agentId: 'my-agent',
    controllerWallet: controllerWallet,
    agentWallet: agentWallet,
    permissions: ['read', 'write'],
  },
  walletAddress: controllerWallet,
});
```

## Agent flow

```
Agent registers -> Controller delegates scope -> Agent acts with receipt
```
