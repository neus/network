Trust infrastructure for agents that act.

NEUS gives agents verifiable identity, scoped authority, and receipts for trusted actions. Use it to prove who controls an agent, what it is allowed to do, and what happened after it acted.

## What NEUS adds

| Need                   | Mechanism                                       |
| ---------------------- | ----------------------------------------------- |
| Identify the agent     | `agent-identity` signed by the agent wallet     |
| Control what it can do | `agent-delegation` with scope, caps, and expiry |
| Check readiness        | `neus_agent_link` in MCP                        |
| Reuse trust in tools   | Stored receipt IDs, proof URLs, wallet, or DID  |
| Audit actions later    | Receipts, tags, and proof reads                 |

## Recommended flow

<Steps>
  <Step title="Register identity">
    Create `agent-identity` for the agent wallet.
  </Step>
  <Step title="Delegate authority">
    Add `agent-delegation` when another user, wallet, or organization controls
    what the agent may do.
  </Step>
  <Step title="Link before creating">
    Call `neus_agent_link` first. If it returns `linked: true`, reuse the
    existing receipts.
  </Step>
  <Step title="Act with receipts">
    Check receipts before tool use, payments, data movement, or other trusted
    actions.
  </Step>
</Steps>

## SDK example

```javascript
await client.verify({
  verifier: "agent-identity",
  data: {
    agentId: "my-agent",
    agentWallet,
    agentLabel: "My AI Agent",
    agentType: "ai",
  },
  walletAddress: agentWallet,
});
```

Add scoped authority with [`agent-delegation`](./agent-delegation).
