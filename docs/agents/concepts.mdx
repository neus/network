Registering the agent and approving what it may do are **two steps**. The publisher and approver can be the same person or not.

## Roles

| Role | What they do | Who signs |
|------|----------------|-----------|
| **Publisher** | Publishes the agent (metadata on the agent wallet) | **Agent wallet** |
| **Approver** | Grants scoped access | **Approving account** |

## Wallets

- **Agent wallet** — The agent’s on-chain identity; used when registering the agent.
- **Approving account** — Signs delegation (scopes, limits, expiry). A different role from the agent wallet.

## Hosted flow

Use **[hosted verify](../cookbook/auth-hosted-verify)** for browser or MCP-guided steps.

## API fields

| Field | Meaning |
|-------|---------|
| `agentWallet` | Agent wallet address |
| `controllerWallet` | Approving account in delegation payloads |
| `linked` (MCP) | Ready: identity and access approval on file |
| `principal` (`neus_me`) | Signed-in NEUS account (DID / wallet) |

Private proofs are visible only to the signed-in user they belong to.
