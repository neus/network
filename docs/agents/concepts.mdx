## Core rule

**Register** and **approve scopes** are separate. Different people can do each. Publisher and approver wallets need not be linked.

## Roles

| Role | Does | Signs |
|------|------|-------|
| **Publisher** | Publishes agent (metadata → agent wallet) | **Agent wallet** |
| **Approver** | Grants scopes | **Approving account** |

Same person or two (marketplace / delegation).

## Wallets

- **Agent wallet:** register / publish.
- **Approving account:** delegation signature (scopes, limits, expiry). Not “owner” of the agent wallet; no profile link required.

## Hosted

**[Hosted verify](../cookbook/auth-hosted-verify)** or MCP links.

## API names

Stable JSON / query names:

| API / query name | Meaning |
|------------------|-------------------------|
| `agentWallet` | Agent wallet address |
| `controllerWallet` | **Approving account** in delegation (not “product controller”) |
| `linked` (MCP) | Readiness: identity + access approval present |
| `principal` (`neus_me`) | Signed-in account summary; not the approver unless same session |

**Private proofs:** only when that user is signed in. Delegation **approving account** is separate.
