## Sequence

<Steps>
 <Step title="Register">
 `agent-identity` - agent wallet signs; one **proof ID** for public metadata.
 </Step>
 <Step title="Delegate (if needed)">
 `agent-delegation` - approving account (`controllerWallet`) signs scope, caps, expiry.
 </Step>
 <Step title="Use in apps & assistants">
 MCP or API for checks, catalogs, hosted verify when browsers are required.
 </Step>
</Steps>

## MCP handoff

`neus_agent_link` to `status: "link_required"`: open **`hostedVerifyUrl`**, finish browser flow, call **`neus_agent_link`** again. Readiness for identity + permissions - not a generic wallet-link tool.
