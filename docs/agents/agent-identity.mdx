Use `agent-identity` to register an AI or automation agent with a reusable proof receipt.

**Verifier ID:** `agent-identity`

## Basic Registration

```javascript
const proof = await client.verify({
  verifier: 'agent-identity',
  data: {
    agentId: 'my-assistant',
    agentWallet: '0x...',
    agentType: 'ai',
    description: 'My AI assistant',
  },
  walletAddress: controllerWallet,
});
```

### Using CAIP-10 Account ID

For cross-chain compatibility, use `agentAccountId`:

```javascript
const proof = await client.verify({
  verifier: 'agent-identity',
  data: {
    agentId: 'my-assistant',
    agentAccountId: 'eip155:8453:0x...',
    agentLabel: 'My Assistant',
    agentType: 'ai',
    description: 'My AI assistant',
  },
  walletAddress: controllerWallet,
});
```

## Full Example

```javascript
await client.verify({
  verifier: 'agent-identity',
  data: {
    agentId: 'my-agent',
    agentWallet: '0x...',           // universal-address: EVM or Solana
    agentType: 'ai',
    description: 'My AI assistant',
    capabilities: ['web-search'],
    instructions: 'System prompt (max 4000 chars)',
    skills: ['web-search', 'code-execution'],   // max 48, each max 64 chars
    services: [                                  // max 16
      { name: 'metrics', endpoint: 'https://metrics.example.com/v1', version: '1.0' }
    ]
  },
  walletAddress: controllerWallet,
});
```

## Fields

| Field | Required | Max Length | Description |
|-------|----------|------------|-------------|
| `agentId` | Yes | 1–128 chars | Unique identifier for the agent |
| `agentWallet` | One of wallet or accountId | — | Agent wallet address (EVM or Solana) |
| `agentAccountId` | One of wallet or accountId | — | CAIP-10 account ID (e.g., `eip155:8453:0x...`) |
| `agentChainRef` | No | — | CAIP-2 chain reference when using `agentWallet` |
| `agentLabel` | No | 128 chars | Human-readable display name |
| `agentType` | No | enum | `ai`, `bot`, `service`, `automation`, or `agent` |
| `description` | No | 500 chars | Agent description |
| `capabilities` | No | 32 items | List of declared capabilities |
| `instructions` | No | 4000 chars | System prompt or behavioral instructions |
| `skills` | No | 48 items | List of skill identifiers |
| `services` | No | 16 items | Service endpoint definitions |

## What You Get Back

You receive a proof receipt ID that can be reused for public agent identity, profile trust, and agent-aware product flows.

## Verification

```bash
GET /api/v1/proofs/check?verifierIds=agent-identity&address=0x...
```
