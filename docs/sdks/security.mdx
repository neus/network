Use the SDK with the same rule as the rest of NEUS: expose the minimum possible data, use hosted verify for interactive work, and keep owner-only proof access behind explicit authorization.

## Authentication Model

| Operation | Authentication |
|-----------|----------------|
| Verification submission (`POST /api/v1/verification`) | Signature over NEUS Standard Signing String |
| Status by proof receipt ID (`GET /api/v1/verification/status/{qHash}`) | Public (no auth required) |
| Owner-only reads of private proofs | Additional owner signature |

Owner-only reads use extra signed headers — call the **SDK methods** for private proofs instead of assembling headers yourself.

## Do Not

<Warning>
- Do not treat proof signatures as bearer tokens (they are request-bound)
- Do not embed **secrets** in browser bundles — keep them on your servers
- Do not log or persist:
  - proof signatures
  - API keys
  - third-party auth credentials or provider tokens
</Warning>

## Recommended Privacy Defaults

```javascript
{
  privacyLevel: 'public',
  publicDisplay: false,
  storeOriginalContent: true
}
```

These are separate controls:

| Control | Purpose |
|---------|---------|
| `privacyLevel` | Controls access |
| `publicDisplay` | Controls public discovery |
| `storeOriginalContent` | Controls whether original content is stored |

<Note>
Default keeps proofs **unlisted** (not discoverable). Use **`private`** when only the **signed-in user** should resolve the proof.
</Note>
