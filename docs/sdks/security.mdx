Less surface area: prefer interactive hosted flows; private reads need owner auth.

## Auth by operation

| Operation | Auth |
|-----------|------|
| `POST /api/v1/verification` | Wallet signature (standard string) |
| `GET /api/v1/proofs/{qHash}` | Public metadata for public/unlisted; private needs owner rules |
| Private payload reads | Owner signature / SDK helpers |

## Do Not

<Warning>
- Do not treat proof signatures as bearer tokens (they are request-bound)
- Do not embed **secrets** in browser bundles
- Do not log or persist:
 - proof signatures
 - API keys
 - third-party auth credentials or provider tokens
</Warning>

## Defaults

`client.verify()` defaults **private** with `storeOriginalContent: true`. VerifyGate create defaults **unlisted public**. [Security and trust](../platform/security-and-trust)

If you use SDK `gateCheck` without owner-authenticated access, set unlisted public explicitly when you need that visibility:

```javascript
{
  privacyLevel: 'public',
  publicDisplay: false,
}
```

Set **`storeOriginalContent: false`** only when your product must **not** persist original bytes (hash/metadata only).

| Control | Purpose |
|---------|---------|
| `privacyLevel` | Vaulted (`private`) vs policy-checkable without owner auth (`public`) |
| `publicDisplay` | Discovery vs unlisted |
| `storeOriginalContent` | Retain original content (default true) vs hash-only |

<Note>
Unlisted public proofs are still public to anyone with the proof id.
</Note>
