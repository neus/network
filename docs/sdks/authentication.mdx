**Default:** no wallet or passkey code in **your** app. Users **sign in** or complete **trust checks** on **`neus.network/verify`** via `VerifyGate`, `getHostedCheckoutUrl`, or MCP OAuth. Credits apply to checks and saved receipts, not to hosted session sign-in.

| Where | What | Wallet code in your app |
|-------|------|---------------------------|
| Browser — access gate | `VerifyGate` + `gateId` | No |
| Browser — sign-in | `getHostedCheckoutUrl` + `intent: 'login'` | No |
| Server | `NeusClient` + `gateCheck` with the same `gateId` | No |
| Automation | `NeusClient` + `npk_*` access key | No |
| Advanced server | `verifyFromApp` after per-user approval | No |
| Browser (exception) | `client.verify({ wallet })` | Yes — [Signing format](../verification/signing-format) |

## VerifyGate (browser)

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

<VerifyGate gateId="gate_your-app-name">
  <ProtectedContent />
</VerifyGate>;
```

## Hosted Verify (browser)

[Hosted Verify](../cookbook/auth-hosted-verify).

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

const url = getHostedCheckoutUrl({
  gateId: 'gate_your-app-name',
  returnUrl: 'https://yourapp.com/auth/callback',
});
```

**Sign-in only** (return to your app after NEUS session):

```javascript
const loginUrl = getHostedCheckoutUrl({
  intent: 'login',
  returnUrl: 'https://yourapp.com/auth/callback',
});
```

## Server reuse

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient();

await client.gateCheck({
  gateId: 'gate_your-app-name',
  address: user.walletAddress,
});
```

## Personal access keys

Pass an access key to `NeusClient` server-side — it is sent as `Authorization: Bearer <npk_...>`:

```javascript
import { NeusClient } from '@neus/sdk';

// Server only — never ship npk_* in a browser bundle
const client = new NeusClient({ apiKey: process.env.NEUS_ACCESS_KEY });
```

Create keys under [Access keys](https://neus.network/profile?tab=account). Server / MCP only.

## IDE and MCP sign-in

For Cursor, VS Code, or Claude Code, run `neus setup`. It configures MCP and starts OAuth for `https://mcp.neus.network/mcp`. This credential is separate from the browser session cookie on `neus.network`. Use a profile access key for servers and CI when browser sign-in is unavailable.

See [MCP setup](../mcp/setup) and [MCP OAuth](../mcp/oauth).

## App link: `verifyFromApp` (advanced server)

After a one-time user approval ([agent delegation app link](../agents/agent-delegation)), your backend can create proofs without a per-request signature. Set the app link on the client and call `verifyFromApp`:

```javascript
const client = new NeusClient({
  apiUrl: 'https://api.neus.network',
  appId: 'acme-web',
  appLinkQHash: process.env.NEUS_APP_LINK_QHASH, // from the user's approval
});

await client.verifyFromApp({
  user: '0x...',
  verifier: 'ownership-basic',
  content: 'Hello NEUS',
  data: { reference: { type: 'url', id: 'https://example.com' } },
});
```

Send `x-neus-app: acme-web` and `Origin: https://yourapp.com` so the delegation's `app:` / `origin:` permissions match.
