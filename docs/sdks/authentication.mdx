NEUS supports both product styles:

- **Bring your own login:** your users stay in your app, and your server calls NEUS through App Link.
- **Hosted NEUS flow:** send users to NEUS when they need hosted login, passkeys, wallet connection, OAuth, human checks, or gate checkout.

## App Link

Use this when your backend already knows the user. Your app keeps its normal login and uses NEUS only when it needs to issue or check a receipt.

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: process.env.NEUS_APP_ID,
  apiKey: process.env.NEUS_ACCESS_KEY,
});

const receipt = await client.verifyFromApp({
  user: { walletAddress: user.walletAddress },
  verifier: 'ownership-basic',
  content: {
    claim: 'User completed onboarding',
    source: 'your-app',
  },
});
```

Use App Link for server-side receipt creation without a wallet prompt on every request.

## Hosted Verify

Use Hosted Verify when NEUS should handle the browser step:

- passkey-first NEUS login
- wallet connection when required
- OAuth or org verification
- human checks
- gate checkout and return handling

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

const url = getHostedCheckoutUrl({
  verifiers: ['ownership-basic'],
  returnUrl: 'https://yourapp.com/neus/callback',
});
```

See [Hosted Verify](../cookbook/auth-hosted-verify).

## Personal Access Keys

Use **`Authorization: Bearer <npk_...>`** from [Access keys](https://neus.network/profile?tab=account) for server-side API calls, jobs, CLI tools, and MCP clients.

Never ship an access key in browser code.

## In-App Wallet Signing

Use direct SDK signing only when your product asks the user to sign with a wallet inside your UI.

```javascript
const client = new NeusClient({ appId: 'your-app-id' });

await client.verify({
  verifier: 'ownership-basic',
  content: 'Hello NEUS',
  wallet: window.ethereum,
});
```

## Raw HTTP Signing

Most teams should not need this. If you implement `POST /api/v1/verification` without the SDK, use [API authentication](../api/authentication) and [Signing format](../verification/signing-format).
