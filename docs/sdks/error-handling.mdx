## When things fail

| Class | Fix |
|-------|-----|
| Validation | Fix request shape |
| Interactive | Hosted `/verify` or VerifyGate |
| Async | Poll with backoff |
| Private read | Owner auth, retry |
| Transient | Backoff; `Retry-After` |

## Polling Example

```javascript
const final = await client.pollProofStatus(proofId, {
  interval: 3000,
  timeout: 60000
});
```

## Private Proof Read Example

```javascript
const privateData = await client.getPrivateProof(proofId, window.ethereum);
```

- Don’t loop-create on ambiguous errors—keep receipt ids for resume.  
- Say when a hosted step is required—avoid generic “error” only.
