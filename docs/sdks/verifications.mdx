Use these patterns to choose how your app creates or reuses proofs. In most products, `reuse-or-create` is the default.

## Privacy and Visibility

The SDK defaults to **private stored** receipts: `privacyLevel: 'private'`, `publicDisplay: false`, `storeOriginalContent: true`. Owner-authenticated reads return full proof data; public/direct reads return only the safe minimal payload unless the caller is authorized.

For **reusable app gates** and server-side `gateCheck` without a signed-in owner, create an **unlisted public** receipt: `privacyLevel: 'public'` with `publicDisplay: false`. Original content still defaults to stored. **Warning:** unlisted is not discoverable in feeds, but it remains **public to anyone with the proof id**—do not use it for secrets.

| Option | When to Use |
|--------|-------------|
| `privacyLevel: 'private'` | Default. Owner or authorized-session access; content vaulted to the wallet. |
| `privacyLevel: 'public'` | Needed for reusable checks without owner session. Pair with `publicDisplay`. |
| `publicDisplay: false` | Unlisted: eligible in checks and direct lookups, not surfaced in discovery. |
| `publicDisplay: true` | Listed on a public profile (requires `privacyLevel: 'public'`). |

### Hide original content

Set `storeOriginalContent: false` when your product should retain only proof metadata/hash/policy status, not original bytes (hash-only mode).

Example (unlisted public, typical for gates):

```javascript
options: {
  privacyLevel: 'public',
  publicDisplay: false,
}
```

## Reuse-First Flow

Use this as the default UX. It minimizes user prompts while still issuing new proofs when needed.

```jsx
<VerifyGate
  requiredVerifiers={['ownership-basic']}
  strategy="reuse-or-create"
  proofOptions={{
    privacyLevel: 'public',
    publicDisplay: false,
  }}
>
  <Protected />
</VerifyGate>
```

## Fresh Proof Flow

Use this when the action is high stakes and you need a newly created assertion.

```jsx
<VerifyGate
  requiredVerifiers={['wallet-risk']}
  strategy="fresh"
>
  <Protected />
</VerifyGate>
```

## Hosted Interactive Flow

Interactive verifiers hand off to NEUS hosted verification (`/verify`).

```jsx
<VerifyGate
  requiredVerifiers={['ownership-social']}
  hostedCheckoutUrl="https://neus.network/verify"
/>
```

## Policy checks from your app

Use the SDK **`gateCheck`** helper instead of downloading full proof payloads for every decision.

```javascript
const result = await client.gateCheck({
  address: '0x...',
  verifierIds: ['token-holding'],
  contractAddress: '0x...',
  minBalance: '1',
  chainId: 8453,
});
```
