```bash
npm install @neus/sdk
```

The JavaScript SDK does four things: build hosted verify URLs, create receipts, poll status, and run server-side gate checks. Use as much or as little as you need — a hosted link and a server check are enough for most apps. MCP calls the same APIs: [MCP](../mcp/overview).

## Default placement

| Use case | Where |
|----------|--------|
| Hosted verify URL, `VerifyGate` | **Browser** (or WebView) |
| **`gateCheck`** with `gateId` | **Server** |
| **`verifyFromApp`**, `npk_*` | **Server** |

Avoid credentialed `fetch` to `api.neus.network` from the browser; use SDK helpers or your backend.

## Browser (default)

Hosted sign-in and checks on **NEUS**:

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

window.location.assign(
  getHostedCheckoutUrl({
    gateId: 'gate_your-app-name',
    returnUrl: 'https://myapp.com/callback',
  }),
);
```

React: `VerifyGate` from `@neus/sdk/widgets` — [Quickstart](../quickstart).

After you have a **receipt ID** (`qHash`), optional wait: `NeusClient` + `pollProofStatus` — [Verifications](./verifications).

## In-app signing (optional)

`client.verify({ wallet: ... })` only if signing stays in **your** page. Pass the wallet/provider explicitly. Non-EVM signing also needs `chain` as CAIP-2. [CAIP-380 Portable Proof](../learn/standards/caip-380) · [Signing format](../verification/signing-format).

## Eligibility on your server

```javascript
const client = new NeusClient();

const gate = await client.gateCheck({
  gateId: 'gate_your-app-name',
  address: '0x...',
});
```

## API mapping

| Need | Method |
|------|--------|
| Hosted handoff | `getHostedCheckoutUrl()`, `VerifyGate` |
| Sign inside your app | `verify()` |
| Wait | `pollProofStatus()` |
| Allow / deny | `gateCheck()` (server) |

Optional: `timeout` on `NeusClient`.

## Defaults

`client.verify()` defaults **private**. [Security](./security). `VerifyGate` create also defaults **private**. [Security and trust](../platform/security-and-trust)
