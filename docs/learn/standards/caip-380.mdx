NEUS tracks the [CAIP-380](https://github.com/ChainAgnostic/CAIPs/pull/380) draft. Draft names may differ; **signing and hashing on NEUS** match production behavior.

## Draft idea

One wallet-signed envelope; apps validate off-chain; stable receipt id.

- Identities: CAIP-10 style (`did:pkh:…`) where used  
- Chains: CAIP-2 (`eip155:1`, `solana:mainnet`, …)  

## NEUS anchor

Receipt = **`qHash`** (0x…): SHAKE-256, 256-bit, over normalized material. SDK often calls the same value **`proofId`**. [Proof receipts](../../platform/receipts-and-results)

## Inputs (conceptual)

Account + chain · `verifierIds` · `data` (deterministic JSON) · `signedTimestamp` · EVM `chainId` or non-EVM `chain`

## Signer string

Six-line UTF-8, LF. Always from `POST /api/v1/verification/standardize` → `signerString` (or SDK `standardizeVerificationRequest`).

```text
NEUS Verification Request
Wallet: <address>
Chain: <chainId or CAIP-2 chain>
Verifiers: <comma-separated-ids>
Data: <json payload>
Timestamp: <unix-ms>
```

Same as [Signing format](../../verification/signing-format).

## Profiles

|  | Chain | Signature |
|--|-------|-----------|
| EVM | `chainId` | EIP-191 / 1271 / 6492 |
| Solana | `solana:…` | Ed25519 |
| Other | CAIP-2 | Profile-specific |

## Freshness

Reject `signedTimestamp` if older than 5 minutes or more than 60 seconds in the future.
