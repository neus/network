**Verifier ID:** `ownership-basic`

Direct-sign (SDK, API, or `VerifyGate` with wallet). Not hosted-only. Visibility: [Security and trust](../platform/security-and-trust).

## Best For

- Content and file provenance
- Authorship and licensing claims
- Portable **verification IDs** (listings, bounties, submissions)

## Flow Type

Instant - signer creates the proof after signing the standardized request.

## What You Configure

| Field | Required | Notes |
|------|----------|-------|
| `owner` | Yes | Must match signer |
| `content` | No | Short human-readable claim |
| `contentHash` | No | Large or external content |
| `reference` | No | URL, IPFS, tx, contract, etc. |
| `provenance` | No | Human / AI / mixed context |

Use `content` for small text; `contentHash` or `reference` for large or off-chain artifacts.

## What You Get Back

Verification id. Reuse for pages, gates, and checks.

## Example

```json
{
  "owner": "0x...",
  "content": "Release notes for v1.2",
  "reference": {
    "type": "url",
    "id": "https://example.com/releases/v1-2"
  }
}
```

## Content retrieval

Resolution needs either stored `publicContent.content` or a stable `reference`. Same signed-in user can resolve private content without extra params. Response includes `contentResolutionStatus`: `resolved` or `hash_only_not_recoverable`.
