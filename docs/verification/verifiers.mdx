Use this page to choose the right verifier family before you drill into the per-verifier pages.

<Note>
The backend verifier registry remains the single source of truth for validation rules and catalog metadata. These docs package the catalog in reader-friendly language.
</Note>

## Choose by Outcome

| What you need | Start with |
|---------------|------------|
| Prove wallet, domain, org, or social ownership | Identity and ownership verifiers |
| Gate by token, NFT, or contract control | Asset and access verifiers |
| Reduce bots or evaluate risk | Trust and risk verifiers |
| Register and control agents | Agent verifiers |

## Flow Types

| Flow type | What it means | Examples |
|-----------|---------------|----------|
| Instant | Signature-driven, completes in one request | ownership-basic, wallet-link, agent-identity |
| Interactive | User must complete a hosted flow first | ownership-social, ownership-org-oauth, proof-of-human |
| Lookup | NEUS checks external or on-chain state | nft-ownership, token-holding, wallet-risk |

## Recommended Bundles

| Bundle | Verifiers | Use case |
|--------|-----------|----------|
| AI trust receipt | ownership-basic + ai-content-moderation + proof-of-human | AI content plus human trust signal |
| Verifiable agents | agent-identity + agent-delegation + wallet-risk | Agents with scoped authority |
| Community gating | nft-ownership or token-holding + proof-of-human | Protected communities and drop access |
| Org onboarding | ownership-org-oauth + ownership-dns-txt + contract-ownership | Team and domain verification |
