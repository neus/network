## Stages

| Stage | Meaning |
|-------|---------|
| Start | Hosted, SDK, or API |
| Accepted | Request in flight |
| Processing | Async verifier |
| Complete | Verification id ready |
| Reused | Checks / gates / MCP |
| Stale / revoked | Policy or user revoke |

## Strategies

| Strategy | Use |
|----------|-----|
| `reuse-or-create` | Default - reuse if valid |
| `reuse` | Read-only checks |
| `fresh` | Force new proof |

## Freshness

- Point-in-time verifiers (balance, risk): respect recency. 
- Prefer cheap checks + `since` / `sinceDays`; verify only when policy needs it.

## Revoke (owner)

```http
POST /api/v1/proofs/revoke-self/{qHash}
```

`qHash` = same value as SDK `proofId`.

```javascript
await client.revokeOwnProof(proofId, wallet);
```
