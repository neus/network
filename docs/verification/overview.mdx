## Loop

```
Check → Verify → Save → Reuse
```

1. **Check** — still valid for policy?  
2. **Verify** — hosted, SDK, or API  
3. **Save** — receipt id  
4. **Reuse** — gates, pages, agents, MCP  

## Outcomes

|  | Examples |
|--|----------|
| Identity | Login, social, team |
| Ownership | Content, files, authorship |
| Access | NFT, token thresholds |
| Abuse | Human, wallet safety |
| Agents | Identity, delegation |

## Verifier types

| Type | How | Examples |
|------|-----|----------|
| Instant | Wallet signs | ownership-basic, agent-identity |
| Interactive | Hosted steps | social, human |
| Lookup | Live state | NFT, balance |

## Terms

| Term | Meaning |
|------|---------|
| Proof receipt id | Stable id — [Receipts](../platform/receipts-and-results) |
| Verifier | Capability checked |
| Signing string | Wallet message |
| Hosted verify | `/verify` UI |

Direct-sign **or** hosted. Visibility: [Security and trust](../platform/security-and-trust)

## Response

- `proofId`  
- `status`: verified · processing · failed  
- Page: `https://neus.network/proof/[proof-id]`
