Verification is how NEUS checks a claim against verifier rules and returns a durable proof id. Read [How it works](./how-it-works) for mechanics and [Integration](../integration) for where verification sits in your app.

## Outcomes

| | Examples |
|--|----------|
| Identity | Login, social, team |
| Ownership | Content, files, authorship |
| Access | NFT, token thresholds |
| Abuse | Human, wallet safety |
| Agents | Identity, delegation |

## Verifier types

| Type | How | Examples |
|------|-----|----------|
| Instant | Wallet signs | ownership-basic, agent-identity |
| Interactive | Hosted steps | social, human |
| Lookup | Live state | NFT, balance |

## Terms

| Term | Meaning |
|------|---------|
| Proof id | Stable id after verify - [Results](../platform/receipts-and-results) |
| Verifier | Capability checked |
| Signing string | Wallet message |
| Hosted Verify | NEUS-hosted browser flow at `/verify` - [guide](../cookbook/auth-hosted-verify) |

**ownership-basic** storage and full text: [Ownership and Provenance](./ownership-basic).

Direct-sign or hosted. Visibility: [Security and trust](../platform/security-and-trust)

## Response

- `proofId`
- `status`: verified, processing, failed
- Page: `https://neus.network/proof/[proof-id]`
