Verification is how NEUS evaluates a claim and returns a **trust receipt** (`qHash`) you can reuse. Mechanics: [How it works](./how-it-works). Placement in your product: [Integration](../integration).

## Outcomes

| | Examples |
|--|----------|
| Identity | Login, social, team |
| Ownership | Content, files, authorship |
| Access | NFT, token thresholds |
| Abuse | Human, wallet safety |
| Agents | Identity, delegation |

## Verifier types

| Type | How | Examples |
|------|-----|----------|
| Instant | Wallet signs | ownership-basic, agent-identity |
| Interactive | Hosted steps | social, human |
| Lookup | Live state | NFT, balance |

## Terms

| Term | Meaning |
|------|---------|
| qHash | Trust receipt reference — [Results](../platform/receipts-and-results) |
| Verifier | Rule set applied to the claim |
| Signing string | Wallet message |
| Hosted Verify | Browser flow at `neus.network/verify` — [Hosted Verify](../cookbook/auth-hosted-verify) |

Visibility: [Security and trust](../platform/security-and-trust)

## Response

- `qHash`
- `status`: verified, processing, failed
- Page: `https://neus.network/proof/[qHash]`
