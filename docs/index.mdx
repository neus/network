**Every app makes you re-verify. NEUS fixes it.**

Each completed verification yields one **proof receipt ID** (`proofId` / `qHash`). Persist it in your product. Use that ID for checks, gates, widgets, and `/proof` links—not raw signatures.

## Lowest-friction order

Do this sequence to avoid billing surprises and rework:

1. **[Hub](https://neus.network/hub)** — sign in with passkey or wallet; review **Credits** and **Apps** when you are close to shipping.
2. **[Get started](./get-started)** — real traffic: **`appId`**, who pays, credits, sponsor vs self-pay.
3. **[Choose your path](./choose-an-integration-path)** — pick Hosted Verify, widgets, SDK, API, or MCP for your UX.
4. **[Integration](./integration)** — production loop: check → verify → save → reuse.

**Prefer no code first?** Use **[Hosted Verify](./cookbook/auth-hosted-verify)** for guided login and interactive checks, or add **[MCP](./mcp/overview)** with the hosted URL only (no key) for catalog and public tools. Use **[Quickstart](./quickstart)** when you already want a minimal SDK proof in dev.

**Assistants:** follow **[LLM docs](./platform/llm-docs)** so answers stay on the lowest-friction path.

## What you can build

| Use case | Verifier |
|----------|----------|
| Human-only signup or access | `proof-of-human` |
| Token / NFT gated products | `nft-ownership` · `token-holding` |
| Verified creator or org badges | `ownership-basic` · `ownership-org-oauth` |
| Proof-backed content and posts | `ownership-basic` |
| Agent identity and delegation | `agent-identity` · `agent-delegation` |
| Domain-verified organizations | `ownership-dns-txt` |
