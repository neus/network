## What is NEUS?

NEUS is the **verification layer for portable trust**.

It turns a completed verification into a **portable proof receipt** you can verify once and reuse across your app, API, access gates, and agent workflows.

<Note>
**Most teams start here:** [Hosted Verify](./cookbook/auth-hosted-verify) for login and interactive verifiers → save the **proof receipt ID** → reuse with checks ([Integration](./integration#loop)) → **your app pays** by default ([Get started](./get-started#choose-who-pays)). SDK, widgets, and MCP are next steps when you outgrow that path—not parallel “first screens.”
</Note>

<Tip>
**Loop:** [Check](./integration#loop) → verify if needed → save **proof receipt ID** → reuse. Interactive flows: **[Hosted Verify](./cookbook/auth-hosted-verify)**. [Proof receipts](./platform/receipts-and-results)
</Tip>
