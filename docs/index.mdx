## How it works

1. **Verify** — User completes verification (hosted UI, wallet, or social).
2. **Save** — You store the proof receipt ID.
3. **Reuse** — Same proof for login, gating, policy, and agents.

## Integration paths

| Path | Best for |
|------|----------|
| [Hosted Verify](./cookbook/auth-hosted-verify) | Login, guided verification, least UI work |
| [Widgets](./widgets/overview) | React gating and badges |
| [SDK](./sdks/overview) | Custom browser or server flows |
| [API](./api/overview) | HTTP from your servers |
| [MCP](./mcp/overview) | Cursor, Claude, agents — browser links first |
