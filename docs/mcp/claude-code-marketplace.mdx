NEUS for Claude Code connects Claude Code to hosted NEUS MCP and adds a small trust-workflow skill for proof receipts, agent identity, delegation, and access-key hygiene.

For Cursor, VS Code, and most local assistants, use [MCP setup](./setup). Claude Code users can use the plugin below when they want the bundled skill.

## What you get

The **`neus-mcp`** plugin includes:

- Hosted NEUS MCP at **`https://mcp.neus.network/mcp`**
- The **`neus-trust-workflow`** skill for receipt reuse, agent flows, and safe account setup

## Install in Claude Code

1. Add the NEUS plugin source:

   ```text
   /plugin marketplace add https://github.com/neus/network
   ```

2. Install NEUS:

   ```text
   /plugin install neus-mcp@neus
   ```

3. Sign in with OAuth (recommended):

   ```bash
   npx -y -p @neus/sdk neus setup
   npx -y -p @neus/sdk neus auth
   npx -y -p @neus/sdk neus doctor --live
   ```

   For servers and automation only:

   ```bash
   npx -y -p @neus/sdk neus setup --access-key <npk_...>
   ```

Create access keys in the NEUS app: [Account -> Access keys](https://neus.network/profile?tab=account).

Check the install:

```bash
npx -y -p @neus/sdk neus doctor
```

Start each session with **`neus_context`**.

## Use the skill

After install, call the skill from Claude Code:

```text
/neus-mcp:neus-trust-workflow
```

## Cursor, VS Code, and custom runtimes

| Surface | MCP | Skills / rules |
| --- | --- | --- |
| **Cursor** | Run **`npx -y -p @neus/sdk neus setup`** ([Setup](./setup)). | Project rules stay in your Cursor workspace. |
| **VS Code** | Run **`neus setup`** ([Setup](./setup)). | No NEUS VS Code extension is required. |
| **Claude Code** | Use **`neus-mcp@neus`** for the skill bundle, then **`neus setup`** and **`neus auth`**. | **`/neus-mcp:neus-trust-workflow`** |
| **OpenAI / custom hosts** | Add the MCP URL — OAuth discovered automatically. | Host-specific. |
