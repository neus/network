**Trust infrastructure for agents that act.**

*Trust that travels.*

Verifiable identity, scoped authority, and receipts for every trusted action — so your assistant knows who is real, what agents may do, and which receipts already exist without re-proving every session.

Agent frameworks handle tools and execution. NEUS handles trust before action: who the agent is, who controls it, what it may do, and what left a receipt.

One OAuth sign-in. The same identity, delegation, and receipts in Cursor, Claude Code, and Codex.

**Endpoint:** `https://mcp.neus.network/mcp`

Signed in on [neus.network](https://neus.network/profile?tab=account)? Use **Open in** for Cursor, Claude Code, or Codex.

## Other editors

The CLI automates install for **Cursor**, **VS Code/Codex**, and **Claude Code** (tabs above). Manual JSON for additional MCP hosts:

<AccordionGroup>
  <Accordion title="Claude Desktop" icon="desktop">

Add remote MCP URL **`https://mcp.neus.network/mcp`** in the app, or under `mcpServers` in `~/.claude.json`.

  </Accordion>

  <Accordion title="Windsurf" icon="wind">

Add to `~/.codeium/windsurf/mcp_config.json`:

```json
{
  "mcpServers": {
    "neus": {
      "url": "https://mcp.neus.network/mcp"
    }
  }
}
```

Restart Windsurf → Cascade → **MCPs**.

  </Accordion>

  <Accordion title="OpenClaw" icon="server">

`~/.openclaw/agents/main/agent/runtime-mcp.json`:

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp",
      "headers": {
        "Authorization": "Bearer <npk_...>"
      }
    }
  }
}
```

Set a Profile access key from [Account → Access keys](https://neus.network/profile?tab=account) as the bearer token in your runtime config.

Trust workflow skill (from a [network repo](https://github.com/neus/network) clone):

```bash
cp -R plugins/neus-trust/skills/neus-trust-workflow ~/.openclaw/workspace/skills/
```

  </Accordion>

  <Accordion title="Hermes" icon="bot">

  <Accordion title="Gemini CLI" icon="sparkles">

`~/.gemini/settings.json`:

## Servers & CI

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

Keys: [Profile → Account](https://neus.network/profile?tab=account). Never paste keys into chat.

## Upgrading from `neus-mcp`

```text
/plugin uninstall neus-mcp@neus
/plugin install neus-trust@neus
```
