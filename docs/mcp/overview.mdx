NEUS MCP gives assistants and agent tools live trust context at **`https://mcp.neus.network/mcp`**.

Use it to check existing receipts, read proof state, link agents, and guide users to hosted verification only when a browser step is required.

## Install

```bash
npx -y -p @neus/sdk neus setup
npx -y -p @neus/sdk neus auth
npx -y -p @neus/sdk neus doctor --live
```

For servers and automation, add a Profile access key:

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

Check setup:

```bash
npx -y -p @neus/sdk neus doctor
```

Full setup: [MCP setup](./setup). OAuth reference: [MCP OAuth](./oauth).

## Session order

| Step | Tool                   | Purpose                                                              |
| ---- | ---------------------- | -------------------------------------------------------------------- |
| 1    | `neus_context`         | Get session context and recommended next steps                       |
| 2    | `neus_me`              | Confirm the Profile key when Bearer auth is configured               |
| 3    | `neus_agent_link`      | Reuse existing agent identity and delegation receipts                |
| 4    | `neus_proofs_check`    | Check eligibility without creating a new receipt                     |
| 5    | `neus_proofs_get`      | Read receipt details when needed                                     |
| 6    | `neus_verify`          | Verify only when signing and profile prerequisites already match     |
| 7    | `neus_verify_or_guide` | Guide the user when setup, payment, wallet, or delegation is missing |

Public tools also include **`neus_secret_create`**, **`neus_secret_list`**, and **`neus_secret_revoke`** when signed in ([Secrets](./secrets)).

## Agent setup

For an agent wallet:

1. Call `neus_agent_link`.
2. If linked, reuse the receipts.
3. If missing, call `neus_agent_create` and complete the returned hosted or signing steps.
4. Call `neus_agent_link` again until `linked: true`.

Details: [Agent verification flow](../agents/agent-verification-flow).

## Import an existing setup

```bash
npx -y -p @neus/sdk neus import --dry-run
npx -y -p @neus/sdk neus import --from auto
```
