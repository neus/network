NEUS MCP is hosted at **`https://mcp.neus.network/mcp`**. It covers receipts, profile-aware flows, eligibility, proof reads, agents, and delegated context ([setup](./setup)).

## Install (editors and assistants)

```bash
npx -y -p @neus/sdk neus setup
```

Optional Profile access key in one step:

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

Verify:

```bash
npx -y -p @neus/sdk neus doctor
```

Per-client paths, manual JSON, and headers: [Setup](./setup).

Recommended flow: reuse existing receipts when possible; open a hosted NEUS URL only when a browser step is required (proof, payment, provider, wallet, profile, or delegation).

## Decision tree (call `neus_context` first every session)

| Step | Action |
|------|--------|
| 1 | **`neus_context`** — session home for profile and context |
| 2 | If Bearer exists, **`neus_me`** |
| 3 | For agents, **`neus_agent_link`** |
| 4 | **`neus_proofs_check`** — check access **without creating** proof records |
| 5 | If satisfied, continue without hosted verify |
| 6 | **`neus_proofs_get`** — proof state, proof inventory, qHash values, tags, optional delegated **`agentWallet`** reads |
| 7 | **`neus_verify`** only when Profile, wallet, and signing already match |
| 8 | **`neus_verify_or_guide`** only when proof, profile, payment, provider, wallet, or delegation setup is missing |

## Delegated agents (MCP)

For an **agent wallet** (automation that signs as its own address), reuse proofs before minting new ones:

1. After **`neus_context`** (and **`neus_me`** when Bearer is configured), call **`neus_agent_link`** with that **`agentWallet`**.
2. If anything is missing, use **`neus_agent_create`** to prepare signing material and a hosted URL, then finish in the browser or with **`neus_verify`** as returned.
3. Call **`neus_agent_link`** again until **`linked: true`**.

Details: [Agent link](./agent-link), [Agent create](./agent-create), and the product-level sequence in [Agent verification flow](../agents/agent-verification-flow).

## Reference calls

- **`neus_verifiers_catalog`:** verifier schemas when **`neus_context`** is not enough.
- **`neus_me`:** after configuring a Profile access key, call once per session (expect **`status: ok`**).

## Headless signing

| Step | What to send |
|------|--------------|
| Prepare | **`neus_verify`** without **`signature`**; sign **`signerString`** |
| Finish | **`neus_verify`** again with the same request fields plus **`signature`** |

Do not repeat prepare.

Field names and tools: [Tools](./tools).
