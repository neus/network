*Trust that travels.*

NEUS MCP connects assistants and tools to live identity, delegation, and receipt state at **`https://mcp.neus.network/mcp`**. Verifiable identity, scoped authority, and receipts for every trusted action - before access, payment, or action. Agent frameworks handle tools and execution; NEUS handles trust before anything runs.

## Install

```bash
npx -y -p @neus/sdk neus setup
npx -y -p @neus/sdk neus auth
npx -y -p @neus/sdk neus doctor --live
```

For servers and automation, add a Profile access key:

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

Check setup:

```bash
npx -y -p @neus/sdk neus doctor
```

Full setup: [MCP setup](./setup). OAuth reference: [MCP OAuth](./oauth).

## Session order

| Step | Tool                   | Purpose                                                              |
| ---- | ---------------------- | -------------------------------------------------------------------- |
| 1    | `neus_context`         | Load current mode, signed-in profile context, verifier summary, and next steps |
| 2    | `neus_agent_link`      | Reuse existing agent identity and delegation receipts                |
| 3    | `neus_proofs_check`    | Check eligibility without creating a new receipt                     |
| 4    | `neus_proofs_get`      | Read receipt details when needed                                     |
| 5    | `neus_verify`          | Verify only when signing and profile prerequisites already match     |
| 6    | `neus_verify_or_guide` | Guide the user when setup, payment, wallet, or delegation is missing |

Use **`neus_me`** only when you need to refresh signed-in profile context or look up a public wallet/DID.

Public tools also include **`neus_secret_create`**, **`neus_secret_list`**, and **`neus_secret_revoke`** when signed in ([Secrets](./secrets)).

## Agent setup

For an agent wallet:

1. Call `neus_agent_link`.
2. If linked, reuse the receipts.
3. If missing, call `neus_agent_create` and complete the returned hosted or signing steps.
4. Call `neus_agent_link` again until `linked: true`.

Details: [Agent verification flow](../agents/agent-verification-flow).

## Import an existing setup

```bash
npx -y -p @neus/sdk neus import --dry-run
npx -y -p @neus/sdk neus import --from auto
```
