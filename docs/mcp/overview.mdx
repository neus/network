NEUS MCP lets an assistant check identity and permissions before sensitive actions. Setup writes the editor configuration for you, and normal editor sign-in does not require an API key.

Connect once with OAuth to use the same profile and trust receipts in Cursor, Claude Code, Codex, and VS Code. Backends can check those same receipts through the API.

## Install

```bash
npm i -g @neus/sdk
neus setup
neus check
```

Or without installing: `npx -y -p @neus/sdk neus setup`

Servers and CI:

```bash
neus setup --access-key <npk_...>
```

Full setup: [MCP setup](./setup). OAuth reference: [MCP OAuth](./oauth).

## How it works

Assistants call **`neus_context`** once to load the signed-in profile, available checks, and recommended order. A valid receipt can be reused in later sessions without repeating the full check.

- **Check before action**: look for an existing receipt, then guide any missing step
- **Show receipts**: read current trust receipt records with **`neus_proofs_get`**
- **Agent readiness**: confirm agent identity and permissions before tools run
- **Connect agent context**: use `neus_agent_mount` or `neus mount` to load an agent into a project
- **Vault**: list, store, or revoke encrypted secrets when signed in

You should see a clear **Passed**, **Action needed**, or **Blocked** summary in chat, not raw tool output.

Journey guide: [MCP journeys](./journeys) · Tool reference: [MCP tools](./tools)

## Agent setup

For an agent wallet:

1. Call `neus_agent_link`.
2. If linked, reuse the receipts.
3. If missing, call `neus_agent_create` and complete the returned hosted or signing steps.
4. Call `neus_agent_link` again until `linked: true`.

Details: [Agent verification flow](../agents/agent-verification-flow) · [Connect Agent Context](../agents/runtime-mount)

## Connect an agent to a project

```bash
neus mount <agentId> --apply cursor
```

Writes `.neus/mount.json` and host rules in the **current project**. Run it again after permission changes.

## Migrate legacy IDE config

```bash
neus import --dry-run
neus import --from auto
neus export --to manifest
```
