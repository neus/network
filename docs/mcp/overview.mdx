MCP is the agent surface. Hosted Verify is the handoff when a user needs to complete a proof.

**Connection URL:** `https://mcp.neus.network/mcp`

Add it once in your MCP client. See [Setup](./setup) for the exact JSON.

## Recommended workflow

1. **Guide the user into verification** - Use `neus_verify_or_guide`. When a tool returns `https://neus.network/verify`, open that link.
2. **Check eligibility** - Use `neus_proofs_check` when you only need a yes or no.
3. **Look up proof history** - Use `neus_proofs_get` when you already have an address or id from NEUS.
4. **Use signed-in identity** - Use `neus_me` after the user is signed in.
5. **Register or link agents** - Use `neus_agent_create` and `neus_agent_link`.

The URL-only install does not sign the chat into NEUS by itself.

| Goal | Tool |
|------|------|
| Send someone to browser verification | `neus_verify_or_guide` |
| Yes/no eligibility only | `neus_proofs_check` |
| Proof list + profile | `neus_proofs_get` |
| Signed-in who am I | `neus_me` |
| Pick verifier IDs safely | `neus_verifiers_catalog` |
| Agents | `neus_agent_create`, `neus_agent_link` |

Always open hosted URLs that tools return. Do not invent extra steps.
