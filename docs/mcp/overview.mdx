**First call:** run **`neus_context`** after you connect. One response covers verifier context, billing model, and protocol rules. **No access key** required.

### Recommended flow

Follow this order unless you have a specific reason not to:

1. **`neus_context`** ‚Äî Live verifier IDs, hub hint, billing hints, constraints (no access key).
2. **Check** ‚Äî **`neus_proofs_check(wallet, verifiers)`** for **yes/no eligibility only** (no browser). Prefer **`neus_verify_or_guide`** when the user may need a browser, passkey, or onboarding.
3. **Hosted verify** ‚Äî If a tool returns **`hostedVerifyUrl`**, open it (`https://neus.network/verify`); do not substitute a different URL.
4. **Proof list** ‚Äî **`neus_proofs_get(identifier)`** for proof records + profile summary.
5. **Account / private vault** ‚Äî Add **`Authorization: Bearer`** (personal access key from **[Access keys](https://neus.network/profile?tab=account)**) only when you need **`neus_me`** or private reads. See **[Auth](./auth)**.
6. **Programmatic signing** ‚Äî **`neus_verify`** without `signature` triggers **one** server **`/standardize`** step and returns **`signerString`** + **`hostedVerifyUrl`**. On the signed follow-up, reuse the **same** `walletAddress`, `verifierIds`, `data`, `signedTimestamp`, and `chain`; you do **not** call **`/standardize`** again. Response may include **`sigVersion`** / **`bodyPreview`** for signing parity.
7. **Agents (advanced)** ‚Äî **`neus_agent_create`** ‚Üí complete hosted steps ‚Üí **`neus_agent_link`** to confirm.

If the API returns **402**, follow **[Billing](../platform/billing)** and **[x402](../platform/x402)** (retry with **`PAYMENT-SIGNATURE`** when your client supports it).

**Summary:** the hosted MCP URL covers public catalog, context, and hosted verify handoffs without a key. Add **`Authorization: Bearer`** (personal access key from **[Access keys](https://neus.network/profile?tab=account)**) only for **`neus_me`**, private reads, or debiting **your** credits ‚Äî **[Auth](./auth)**. ## Workflow

Add the MCP endpoint **once** in your client‚Äîsee **[Setup](./setup)** for URL-only and authenticated configuration blocks.

1. **`neus_context`** ‚Äî load live verifier catalog and protocol context (public).
2. **Browser verify:** assistant runs **`neus_verify_or_guide`** or **`neus_proofs_check`**. User opens **`https://neus.network/verify`** from the tool. Passkey or wallet stays in NEUS (no pasted one-time codes in MCP).
3. **Signed-in assistant:** add **Bearer** with your **personal access key**, then **`neus_me`** when you need the current principal. **[Auth](./auth)**
4. **Proof history:** when you already have an address or id, **`neus_proofs_get`**. [Tools](./tools)

| Goal | Tool |
|------|------|
| Verifiers, billing model, protocol rules (call first) | `neus_context` |
| Send someone to browser verification | `neus_verify_or_guide` |
| Yes/no eligibility only | `neus_proofs_check` |
| Proof list + profile (you have an id) | `neus_proofs_get` |
| Signed-in ‚Äúwho am I‚Äù | `neus_me` ¬∑ Bearer ¬∑ [Auth](./auth) |
| Pick verifier IDs safely | `neus_verifiers_catalog` |
| Agents | `neus_agent_create`, `neus_agent_link` |

Open hosted URLs from tools.
