**First call:** `neus_context` after you connect — verifier context, billing hints, constraints. No access key required.

### Proof vs runtime

Skill refs on an agent proof are declarative caps, not secrets or executable tools. Your MCP host supplies credentials and execution; keep sensitive config out of proof payloads. **≤48** skill refs per proof.

### Flow

1. **`neus_context`**
2. **`neus_proofs_check`** — yes/no only; or **`neus_verify_or_guide`** if the user may need a browser / passkey
3. Open **`hostedVerifyUrl`** as `https://neus.network/verify` when returned
4. **`neus_proofs_get`** when you have an identifier
5. **`Authorization: Bearer`** ([Access keys](https://neus.network/profile?tab=account)) for **`neus_me`** and vaulted reads — [Auth](./auth)
6. **`neus_verify`** unsigned: one `/standardize`; reuse the same payload on the signed call (no second `/standardize`)
7. Agents: **`neus_agent_create`** → hosted steps → **`neus_agent_link`**

**Summary:** hosted URL covers public tools without a key. Bearer for account-scoped tools and credits. [Get started](../get-started) · [Billing](../platform/billing) · [Setup](./setup).

## Workflow

[Setup](./setup) — drop the URL into your client.

1. **`neus_context`**
2. **`neus_verify_or_guide`** / **`neus_proofs_check`** → user hits **`https://neus.network/verify`** when needed
3. Bearer + **`neus_me`** when signed-in context matters — [Auth](./auth)
4. **`neus_proofs_get`** for history — [Tools](./tools)

| Goal | Tool |
| --- | --- |
| Context (call first) | `neus_context` |
| Browser verification | `neus_verify_or_guide` |
| Eligibility only | `neus_proofs_check` |
| Proof list + profile | `neus_proofs_get` |
| Current principal | `neus_me` + Bearer |
| Safe ID pick | `neus_verifiers_catalog` |
| Agents | `neus_agent_create`, `neus_agent_link` |
