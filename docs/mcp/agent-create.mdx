Use this tool **after** **`neus_agent_link`** shows missing proofs (or when you are bootstrapping a brand-new agent wallet and already know you need both steps). It **does not** store receipts by itself: it returns **`signatures`** and/or a **`hostedVerifyUrl`** for you to complete, then you confirm with **`neus_agent_link`**.

| Step | Action |
|------|--------|
| 1 | **`neus_agent_link`** on the **`agentWallet`** when that address already exists in your design |
| 2 | If proofs are missing, call **`neus_agent_create`** with **`agentId`** and **`agentWallet`** (see below) |
| 3 | Complete **`signatures`** (two steps: agent wallet, then approving account) **or** open **`hostedVerifyUrl`** once |
| 4 | Call **`neus_agent_link`** again until **`linked: true`** |

Optional **`controllerWallet`**: omit when the signed-in account from **`neus_context`** should approve the delegation.

Vocabulary: [Agent concepts](../agents/concepts).

## Agent wallet

| Value | When to use |
|-------|-------------|
| **EVM / Solana address** | You custody the agent key (recommended for teams that reuse one automation wallet) |
| **`"generate"`** | NEUS generates a new agent wallet for advanced runtime setup. Store returned key material only in your secrets manager. |

## Minimal input (existing agent wallet)

```json
{ "agentId": "my-assistant", "agentWallet": "0x1234567890abcdef1234567890abcdef12345678" }
```

Optional **`instructions`**, **`skills`**, **`services`**, presets, spend caps, and delegation fields: [Agent identity](../agents/agent-identity), [Agent delegation](../agents/agent-delegation), and the SDK.
