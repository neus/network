## Session-Backed Use

Set `NEUS_AUTH_TOKEN` in the MCP server environment when you want session-backed calls:

- **`neus_me`** — With no `identifier`, resolves the principal through `GET /api/v1/auth/me`.
- **Private proof access** — Eligibility checks remain eligibility-safe; the token is used for authenticated API calls where your integration has been granted access.

<Warning>
Do not run a publicly reachable MCP server with privileged API credentials. The hosted NEUS MCP service is transport-open by design; signer checks, payment policy, and rate limits are enforced by the NEUS API.
</Warning>

## Eligibility-Safe Behavior

- Gate checks are **eligibility-safe**. They do not enumerate receipts.
- Private proof payloads are **not exposed** through the public MCP tools.
- The server sends `X-Neus-App` using `NEUS_MCP_APP_ID` for attribution.
- Sponsor grants can be forwarded through `neus_proofs_check`.
- x402 payment-required responses are passed through in a structured way instead of being swallowed.
- HTTP mode supports optional Bearer auth plus origin/host allowlists for local HTTP instances.

## Auth-Required Responses

When a tool needs authentication and none is present, the server can return:

- `status: "auth_required"` with a hosted login URL for the user to complete sign-in.
- Use the returned URL to hand off to `https://neus.network/verify` (or your configured hosted verify base).
