Use the hosted MCP endpoint unless you have a separate reason to run your own transport.

## Client Setup

### Cursor

```json
{
  "mcpServers": {
    "neus": {
      "type": "streamableHttp",
      "url": "https://mcp.neus.network/mcp",
      "description": "NEUS MCP (live)"
    }
  }
}
```

## Integration Quick Start

1. **Add the MCP server** to your client config (examples above).
2. **Call `neus_context`** to get the verifier catalog, billing model, and constraints in one call.
3. **Use eligibility checks** (`neus_proofs_check`) to gate access without loading full proof data.
4. **For personalized context**, authenticate via `https://neus.network/verify?intent=login` and set `NEUS_AUTH_TOKEN` in your environment.
5. **Reference proofs by qHash** — the proof receipt ID works across all chains and protocols.

## What You Get

| Capability | Tool | Use Case |
|------------|------|----------|
| Verifier catalog | `neus_verifiers_catalog` | Discover available verification types |
| Eligibility checks | `neus_proofs_check` | Gate access without user interaction |
| Proof lookup | `neus_proofs_get` | Get proof history for a wallet/DID |
| Agent verification | `neus_agent_link` | Check agent identity and delegation |
| Guided onboarding | `neus_verify_or_guide` | Get hosted verify URL when proofs missing |
