Shipping a product? [Get started](../get-started) first.

## URL only

Paste into MCP config (Cursor, Claude Code, Desktop, VS Code, …). Keep `/mcp` in the URL.

```json
{
  "mcpServers": {
    "neus": {
      "type": "streamableHttp",
      "url": "https://mcp.neus.network/mcp",
      "description": "NEUS MCP"
    }
  }
}
```

Reload if needed. URL-only is enough for `neus_context`, catalog, and verify links.

## First call

`neus_context` once after connect.

## Account access (optional)

For `neus_me`, private reads, or **your** credits: add a key from [Access keys](https://neus.network/profile?tab=account) to MCP headers. [Auth](./auth)

Never in browser bundles. Prod apps: [Get started](../get-started), not your personal key.

### Example: Cursor (`mcp.json`)

`mcp.json`:

```json
{
  "mcpServers": {
    "neus": {
      "type": "streamableHttp",
      "url": "https://mcp.neus.network/mcp",
      "headers": { "Authorization": "Bearer <key>" }
    }
  }
}
```

### Example: Claude Code

Use the same JSON block in your client’s MCP settings (see that product’s **MCP over HTTP** guide).

### Other clients

Follow your client’s **MCP over HTTP** setup: paste the URL first; add headers when supported.

## First run

1. `neus_context`
2. `neus_verify_or_guide` or `neus_proofs_check`
3. Open `https://neus.network/verify` when given
4. `neus_proofs_get` if needed - [Tools](./tools)
5. Access key for account tools - [Auth](./auth)

Then [Overview](./overview).
