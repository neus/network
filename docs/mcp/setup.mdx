Use the hosted MCP endpoint unless you have a separate reason to run your own transport.

## Client Setup

### Cursor

```json
{
  "mcpServers": {
    "neus": {
      "type": "streamableHttp",
      "url": "https://mcp.neus.network/mcp",
      "description": "NEUS MCP (live)"
    }
  }
}
```

## Recommended Go-Live Pattern

For production integrations, keep the integration model simple:

1. Use the hosted MCP endpoint for fastest production adoption.
2. Let users try it immediately without transport auth.
3. When personalized context is needed, send users to `https://neus.network/verify?intent=login`.
4. Reuse the same NEUS session outside the browser by setting `NEUS_AUTH_TOKEN`.
5. Start personalized bootstrap with `neus_me`, then `neus_agent_link`, then `neus_proofs_get`, then `neus_verifiers_catalog`.
6. Keep private proof handling inside NEUS; consume eligibility and receipt IDs, not raw claims.
