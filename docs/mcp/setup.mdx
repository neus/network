Hosted NEUS MCP runs at **`https://mcp.neus.network/mcp`**.

## One-command setup

```bash
npx -y -p @neus/sdk neus setup
```

Then sign in to enable account-aware tools:

```bash
npx -y -p @neus/sdk neus auth
```

For servers and automation, add your key directly:

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

Check the install:

```bash
npx -y -p @neus/sdk neus doctor
```

Run a live hosted MCP check:

```bash
npx -y -p @neus/sdk neus doctor --live --access-key <npk_...>
```

Create keys under **Profile -> Account** on [neus.network](https://neus.network/profile?tab=account). Treat access keys like passwords. Keep them in MCP or server config only.

## Import an existing agent setup

```bash
npx -y -p @neus/sdk neus import --dry-run
npx -y -p @neus/sdk neus import --from auto
npx -y -p @neus/sdk neus export --to manifest
```

`neus import` detects OpenClaw, HERMES, Cursor rules, Claude Code skills/settings, and Claude Desktop MCP config. It writes portable metadata to `.neus/imported/` when not in dry-run mode. Secret-like env names are recorded as references only; values are not written.

## Where config is written

| Client      | User scope                                                                          |
| ----------- | ----------------------------------------------------------------------------------- |
| Cursor      | `~/.cursor/mcp.json`                                                                |
| VS Code     | VS Code User `mcp.json`                                                             |
| Claude Code | User scope through `claude mcp add`; project `.mcp.json` with `neus init --project` |

There is no separate NEUS editor plugin required for Cursor or VS Code.

## Manual config

Keep `/mcp` in the URL.

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp"
    }
  }
}
```

With a Profile access key:

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp",
      "headers": { "Authorization": "Bearer <key>" }
    }
  }
}
```

## First session

Start with `neus_context`. If Bearer auth is configured, call `neus_me` once, then reuse receipts with `neus_proofs_check` and read exact receipt records with `neus_proofs_get` before starting new verification.
