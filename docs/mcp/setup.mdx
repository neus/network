**Building on NEUS?** Do **[Get started](../get-started)** first so platform setup, **`appId`**, and credits match how you use MCP in production.

## Connect

Add this configuration to your MCP settings (Cursor, Claude Desktop, VS Code, or any client that supports HTTP MCP). Use the URL exactly as shown, including `/mcp`.

```json
{
  "mcpServers": {
    "neus": {
      "type": "streamableHttp",
      "url": "https://mcp.neus.network/mcp",
      "description": "NEUS MCP"
    }
  }
}
```

Reload the client if prompted. **You only need the URL** to use public discovery and verification flows. Add **`Authorization: Bearer`** with your **personal API key** when you need a signed-in session or private data ([Auth](./auth)); create the key on **[Access keys](https://neus.network/profile?tab=account)**.

<Tip>
  **Recommended:** connect with the URL first. If a tool returns **`auth_required`** and account context is required, add **`Authorization: Bearer`** using a personal API key from Access keys. Public catalog and verification flows do not require a key.
</Tip>

Hosted verification links work for both web2 and web3 users: open the link, follow the NEUS flow, and only connect/sign when the selected proof actually requires it.

## First run

1. Ask the assistant to run **`neus_verify_or_guide`** (or **`neus_proofs_check`**).
2. **Open** any **`https://neus.network/verify`** link in the browser.
3. Optional **`neus_proofs_get`** when you have an id. [Tools](./tools)
4. Sessions / private data: [Auth](./auth).

Then follow the [Overview](./overview) default path.

## Development endpoints

If your integration targets **`https://dev.api.neus.network`**, use **`https://dev.mcp.neus.network/mcp`** for MCP. [Endpoints](./endpoints)
