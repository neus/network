Connect your editor or agent to the hosted NEUS MCP endpoint. Reuse existing receipts when possible. Use hosted URLs when setup or proof state is missing.

## Universal install (any IDE or agent host)

Use **`@neus/sdk`** as the single npm entrypoint for every desktop editor. Same URL, same optional Profile key, same **`neus doctor`** check. **Do not fork** parallel installers per IDE — that creates drift.

| Concern | Golden path |
| --- | --- |
| MCP wiring | `npx -y -p @neus/sdk neus setup` (or `neus init` without keys) |
| Trust / “is it working?” | `npx -y -p @neus/sdk neus doctor` |
| Claude-only discoverability | Optional [NEUS for Claude Code](./claude-code-marketplace) plugin (skill + default MCP); still run **`neus setup`** for keys |
| OpenClaw / containers | Env + `NEUS_MCP_URL` in your runtime — same MCP endpoint and tool contract |

## One-command setup

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

`setup` runs `init` and `auth` in one pass on every detected client (Claude, Cursor, VS Code). Omit `--access-key` when you only want the base MCP config without personal account tools.

## Where config is written

After `neus setup` or `neus init`, the CLI merges NEUS into each detected client:

| Client | User scope (default) |
| --- | --- |
| Cursor | `~/.cursor/mcp.json` (`mcpServers.neus`) |
| VS Code | VS Code **User** `mcp.json` (path varies by OS) |
| Claude Code | User scope via `claude mcp add`, or project `.mcp.json` with `neus init --project` |

There is **no separate NEUS Cursor plugin** in this repo: Cursor uses the same command and URL as everyone else.

## Claude Code marketplace (optional)

To install hosted MCP plus the bundled trust-workflow skill from the git catalog, see [NEUS for Claude Code](./claude-code-marketplace). Run `neus setup` afterward for your access key so Cursor/VS Code/Claude stay consistent.

## Step-by-step alternative

```bash
npx -y -p @neus/sdk neus init
```

That command configures NEUS MCP in supported clients automatically. Add `--project` when you want shared repo config. Add `--client claude`, `--client cursor`, or `--client vscode` when you want to target one client directly. Add personal auth later with:

```bash
npx -y -p @neus/sdk neus auth --access-key <npk_...>
```

Confirm the final state with:

```bash
npx -y -p @neus/sdk neus status --json
```


## Config

Keep **`/mcp`** in the URL.

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp"
    }
  }
}
```

## Optional Bearer

`Authorization: Bearer` with **`npk_*`** enables **`neus_me`**, private reads, and signatureless **`neus_verify`** when the profile wallet matches. See [Auth](./auth).

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp",
      "headers": { "Authorization": "Bearer <key>" }
    }
  }
}
```

| Check | |
|-------|--|
| Keys | [Access keys](https://neus.network/profile?tab=account) |
| Confirm | **`neus_me`** → **`status: ok`** |
| Never | Ship keys to the browser |

## First run

| # | Step |
|---|------|
| 1 | Connect (config above) |
| 2 | **`neus_context`** first |
| 3 | If Bearer exists, **`neus_me`** |
| 4 | For agents, **`neus_agent_link`** |
| 5 | Check with **`neus_proofs_check`** |
| 6 | If satisfied, continue without hosted verify |
| 7 | Use **`neus_proofs_get`** when you need proof state, inventory, tags, or qHash values |
| 8 | Use **`neus_verify_or_guide`** only when something is missing |

Other clients: same URL, same optional **`Authorization`** header.

## Doctor check

```bash
npx -y -p @neus/sdk neus doctor --access-key <npk_...>
```

`doctor` confirms config health, profile connection, and agent verification readiness across every configured client. Run `doctor --json` for machine-readable output. Omit `--access-key` when you want config-only status.
