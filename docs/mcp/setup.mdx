Use the hosted MCP endpoint unless you have a separate reason to run your own transport.

## Recommended Go-Live Setup

| Surface | Use this when | Setup |
|---------|---------------|-------|
| Hosted MCP | You want the public NEUS MCP immediately | `https://mcp.neus.network/mcp` |
| Plugin install | You want packaged Zeus + NEUS skills + hosted MCP wiring | `github:neus/network/plugins/neus-core` |
| Local stdio | You are testing or self-hosting the MCP package | `npx @neus/mcp-server` or `node mcp/server.js` |

## Plugin Install

### Claude Code / Cursor plugin

```bash
/plugin install github:neus/network/plugins/neus-core
```

The `neus-core` plugin points to the hosted MCP endpoint and includes the Zeus agent plus the bundled NEUS skills.

## Integration Quick Start

1. Add the MCP server to your client config.
2. Call `neus_context` to load verifiers, billing model, and constraints.
3. Use `neus_proofs_check` or `neus_verify_or_guide` before trying to create new proofs.
4. For personalized context, authenticate via `https://neus.network/verify?intent=login` and set `NEUS_AUTH_TOKEN`.
5. Reference proofs by qHash. The proof receipt ID works across all chains and protocols.

## What You Get

| Capability | Tool | Use case |
|------------|------|----------|
| Full session bootstrap | `neus_context` | Load the live contract before any gating or agent flow |
| Verifier catalog | `neus_verifiers_catalog` | Discover available verification types |
| Direct proof creation | `neus_verify` | Return elicitation or create the proof immediately |
| Eligibility checks | `neus_proofs_check` | Gate access without user interaction |
| Proof lookup | `neus_proofs_get` | Get proof history for a wallet/DID |
| Agent verification | `neus_agent_link` | Check agent identity and delegation |
| Agent creation | `neus_agent_create` | Prepare the two-signature identity + delegation flow |
| Guided onboarding | `neus_verify_or_guide` | Get hosted verify URL when proofs are missing |

## Auth Notes

- No API key is required to connect to the hosted MCP endpoint.
- `NEUS_AUTH_TOKEN` is only needed when you want the same NEUS session in non-browser contexts.
- Private proof payloads do not become public just because you are using MCP.
