Hosted NEUS MCP runs at **`https://mcp.neus.network/mcp`**.

Production-stable for production apps; [public beta](../platform/status) until mainnet and planned security audits.

## One-command setup

Install the hosted MCP endpoint, then sign in:

```bash
npx -y -p @neus/sdk neus setup
npx -y -p @neus/sdk neus auth
npx -y -p @neus/sdk neus doctor --live
```

For servers and automation, add your key in one step:

```bash
npx -y -p @neus/sdk neus setup --access-key <npk_...>
```

Check the install anytime:

```bash
npx -y -p @neus/sdk neus doctor
```

Create keys under **Profile -> Account** on [neus.network](https://neus.network/profile?tab=account) for servers and CI only. Treat access keys like passwords.

## npm packages

| Package | Use |
| ------- | --- |
| **`@neus/sdk`** | SDK, widgets, and `neus` CLI (`setup`, `auth`, import/export) |
| **`@neus/mcp-server`** | Registry/discovery metadata (`server.json`) for MCP installers — no runtime |

```bash
npm install @neus/mcp-server
```

## Disconnect

```bash
npx -y -p @neus/sdk neus disconnect --access-key <token>
```

## Import an existing agent setup

```bash
npx -y -p @neus/sdk neus import --dry-run
npx -y -p @neus/sdk neus import --from auto
npx -y -p @neus/sdk neus export --to manifest
```

## Client-specific setup

### Claude Code

Claude Code supports MCP OAuth discovery. When you add a remote MCP server, it automatically detects the NEUS OAuth endpoints and starts a browser-based login flow.

```bash
claude mcp add --transport http neus https://mcp.neus.network/mcp
```

Or use the one-command setup:

```bash
npx -y -p @neus/sdk neus setup
```

### OpenAI / ChatGPT

Add NEUS as a remote MCP server app. OpenAI discovers the OAuth metadata and presents the NEUS authorization flow.

### VS Code / Cursor

Add the MCP URL to your configuration. VS Code and Cursor read OAuth metadata from `https://mcp.neus.network/.well-known/mcp.json` and can run browser sign-in without pasting keys.

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp"
    }
  }
}
```

Or run the CLI (recommended):

```bash
npx -y -p @neus/sdk neus setup
npx -y -p @neus/sdk neus auth
```

If OAuth is not supported by your client version, use a Profile access key as fallback:

```json
{
  "mcpServers": {
    "neus": {
      "type": "http",
      "url": "https://mcp.neus.network/mcp",
      "headers": { "Authorization": "Bearer <npk_...>" }
    }
  }
}
```

## Where config is written

| Client      | User scope                                                                          |
| ----------- | ----------------------------------------------------------------------------------- |
| Cursor      | `~/.cursor/mcp.json`                                                                |
| VS Code     | VS Code User `mcp.json`                                                             |
| Claude Code | User scope through `claude mcp add`; project `.mcp.json` with `neus setup --project` |

There is no separate NEUS editor plugin required for Cursor or VS Code.

## First session

Start with `neus_context`. If Bearer auth is configured, call `neus_me` once, then reuse receipts with `neus_proofs_check` and read exact receipt records with `neus_proofs_get` before starting new verification.
