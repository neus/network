title: Proof Context (`neus_proofs_get`)
description: Proofs and profile summary for a wallet or DID.
icon: receipt
---

Use when you have an **`identifier`**. If the user still needs to verify, **`neus_verify_or_guide`** first.

**Visibility:** no Bearer → public-scope view; Bearer → can include **private** proofs for that account ([Auth](./auth)).

## Input

```json
{
  "identifier": "0x…",
  "limit": 25,
  "offset": 0,
  "tags": "memory,skill,job",
  "agentWallet": "0x…"
}
```

**`identifier`:** EVM, Solana base58, **`did:pkh:…`**. **`tags`:** comma-separated filter on **`proof.meta.tags`**. **`agentWallet`:** delegation (**`x-agent-wallet`**).

## Response

Proofs, **`profileSummary`**, pagination: follow the live tool response. **Tags** and proof bodies come from this call.
