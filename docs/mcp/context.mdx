Call **`neus_context`** once at the start of a NEUS MCP session.

It returns the current mode, setup links, session flow, safety rules, compact verifier summary, and **signed-in profile context when the MCP connection is authenticated**.

Use **`neus_me`** only when you need to refresh profile context or look up a public wallet/DID.

Authenticated profile context can include `profileSummary.proofsSummary`: a compact receipt snapshot with total counts, verifier counts, and latest receipt pointers. Use **`neus_proofs_get`** when you need full receipt records or pagination.

## What it returns

| Field | Use |
|-------|-----|
| `setup.mcpEndpoint` | The hosted MCP endpoint in use (`https://mcp.neus.network/mcp`) |
| `sessionWallet` | Signed-in wallet — omit `wallet` on check/verify tools when present |
| `runtimeContract` | Canonical intent → tool-sequence map plus execution and stop rules — follow it instead of improvising tool order |
| `autopilot` | Recommended trust-before-action flow |
| `verifierSummary` | Compact list of available checks |
| `trustResult` | Current trust posture for the session |
| `tools` | Map of available NEUS MCP tools |
| `profileSummary` | Signed-in profile context, including `proofsSummary` |

When signed in, prefer `sessionWallet` and reuse existing receipts before opening any hosted link.
