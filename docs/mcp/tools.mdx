Each MCP tool does one job. Start here, then move into the task-specific page that matches your flow.

## Recommended Order

1. `neus_context`
2. `neus_verifiers_catalog` when you need detailed live verifier metadata
3. `neus_proofs_check` or `neus_verify_or_guide`
4. `neus_verify` only when you are intentionally creating a proof
5. `neus_agent_create` only when you are intentionally creating a new agent

## Full Tool Contract

| Tool | Contract |
|------|----------|
| `neus_context` | Call first. Returns verifiers, billing, protocol rules, and agent constraints in one response. |
| `neus_verifiers_catalog` | Read the live verifier catalog. |
| `neus_proofs_check` | Eligibility-safe proof check only. |
| `neus_verify_or_guide` | Gate-first orchestration with hosted verify handoff. |
| `neus_verify` | Low-level proof creation or signing elicitation. |
| `neus_proofs_get` | Public proof context, profile summary, and agent context pack. |
| `neus_me` | Session-backed principal context when `NEUS_AUTH_TOKEN` is present. |
| `neus_agent_link` | Check whether agent identity + delegation are both present. |
| `neus_agent_create` | Prepare the two-step signature flow to create a new agent. |
