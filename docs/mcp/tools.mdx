`neus_proofs_check` can take `sponsorGrant` if you use sponsors ([Get started](../get-started)). You mostly **open** verify URLs the model returns.

Bearer: [Auth](./auth)

## Field names

| Tool | Address | Verifiers |
|------|---------|-----------|
| `neus_proofs_check` | `wallet` | `verifiers` |
| `neus_verify_or_guide`, `neus_verify` | `walletAddress` | `verifierIds` |
| `neus_proofs_get` | `identifier` | — |

EVM `0x…`, Solana base58, or `did:pkh:…` as applicable.

## Order

1. `neus_context`
2. `neus_verify_or_guide` / `neus_proofs_check`
3. `neus_verifiers_catalog` (ids)
4. `neus_proofs_get` (have id)
5. `neus_me` (Bearer — [Auth](./auth))
6. Agent tools if needed
