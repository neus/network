After `neus setup` and `neus check`, talk to your assistant in plain language. It reports **Passed**, **Action needed**, or **Blocked**. Not raw tool output.

When you are signed in, tools use your current profile automatically.

## Before a sensitive action

Reuse a saved result when one fits. Run a new check only when needed.

**Try:** "Before I take a sensitive action, use NEUS. Reuse what I already have."

```txt
NEUS: Passed. Requirement satisfied. Result on file. Next: Continue.
```

If nothing on file covers the request:

```txt
NEUS: Action needed. Missing: Sign-in. Next: Complete the secure step, then retry.
```

## Show saved results

Pull the URL, status, tags, or summary behind any prior check.

**Try:** "Show my NEUS results for identity checks."

## Carry context forward

Tagged results carry prior instructions, rules, or session references so your assistant picks up where you left off. Optional patterns: [Receipt links and versioning](../learn/receipts-as-knowledge).

**Try:** "Use my NEUS context from prior results before asking me again."

## Confirm an agent before tools run

Confirm the agent has identity and approved authority before it acts.

**Try:** "Confirm this agent is allowed to run tools before it acts."

```txt
NEUS: Blocked. A required condition was not satisfied. Next: Do not continue until it passes.
```

When setup is missing, the assistant guides identity and authority, then re-checks until the agent is linked. See [Agent verification flow](../agents/agent-verification-flow).

## Pay per call within a cap

With a payments delegation and a spend cap, an agent can settle metered calls and is refused past the cap.

**Try:** "Pay for this API call with my NEUS payment authority, up to the cap."

Details: [Pay per call](../platform/x402).

## Store secrets safely

Keep tokens in encrypted Vault instead of pasting them into chat.

**Try:** "Use NEUS Vault before storing or using secrets."

Details: [Secrets](./secrets).
