title: Verify or Guide (`neus_verify_or_guide`)
description: Fallback setup when reuse checks cannot satisfy the request.

---

Use after **`neus_context`**, **`neus_agent_link`** for agents, and **`neus_proofs_check`** for the required verifier IDs.

If existing receipts satisfy the request, continue without this tool. Use **`neus_verify_or_guide`** only when proof, profile, payment, wallet, or delegation setup is missing.

Returns an **`action`** (`already_verified` or `verification_required`) and includes **`hostedVerifyUrl`** when a hosted step is required. After hosted completion, call **`neus_proofs_get`** on the same identifier to read proof state and receipt fields ([Overview](./overview)).

## Input

`walletAddress` (EVM / Solana / DID-shaped), `verifierIds`, optional `requireAll`, `chain`, `data`.

```json
{
  "walletAddress": "0x…",
  "verifierIds": ["proof-of-human"],
  "requireAll": true
}
```

## Output

| Result | Meaning |
|--------|---------|
| **`already_verified`** | Policy satisfied; no browser step needed |
| **`verification_required`** | Open **`hostedVerifyUrl`**; honor **`requiredVerifiers`** |

Schemas load from the server. Auth: [Auth](./auth).
