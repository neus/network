---
title: Verify or Guide (`neus_verify_or_guide`)
description: Check whether verification is already satisfied and return hosted guidance when it is not.
icon: route
---

Use `neus_verify_or_guide` when you want one MCP tool to decide whether the user is already eligible or should be sent into hosted verify.

## Best For

- agent assistants
- guided onboarding
- hosted verify handoff

## Input

```json
{
  "walletAddress": "0xabc123...",
  "verifierIds": ["proof-of-human", "wallet-risk"],
  "requireAll": true
}
```

## Returns

- `action: "already_verified"` when the policy is already satisfied
- `action: "verification_required"` plus `hostedVerifyUrl` when the user must verify

## Response Structure

### Already Verified

```json
{
  "success": true,
  "action": "already_verified",
  "eligible": true,
  "matchedCount": 2,
  "message": "Proof requirements already satisfied"
}
```

### Verification Required

```json
{
  "success": false,
  "status": "verification_required",
  "action": "verification_required",
  "hostedVerifyUrl": "https://neus.network/verify?verifiers=proof-of-human,wallet-risk",
  "matchedCount": 0,
  "nextSteps": {
    "tool": "neus_verify",
    "requiredVerifiers": ["proof-of-human", "wallet-risk"]
  }
}
```
