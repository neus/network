---
title: Verify or Guide (`neus_verify_or_guide`)
description: Already verified, or open a hosted NEUS verify link.
icon: signs-post
---

Returns **`already_verified`** or **`hostedVerifyUrl`**. User completes in the browser. Then **`neus_proofs_get`** on the same identifier to confirm **`qHash`** ([Overview](./overview)).

## Input

`walletAddress` (EVM / Solana / DID-shaped), `verifierIds`, optional `requireAll`, `chain`, `data`.

```json
{
  "walletAddress": "0x…",
  "verifierIds": ["proof-of-human"],
  "requireAll": true
}
```

## Output

- **`already_verified`:** policy satisfied.
- **`verification_required`:** open **`hostedVerifyUrl`**; honor **`requiredVerifiers`**.

Schemas load from the server. Bearer / **`neus_me`**: [Auth](./auth).
