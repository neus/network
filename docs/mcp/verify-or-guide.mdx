**Default “prove it” tool:** returns **`already_verified`** or a **`hostedVerifyUrl`** — user finishes in the browser; no extra steps in chat.

## Input

`walletAddress` (EVM / Solana / DID-shaped), `verifierIds`, optional `requireAll`, `chain`, `data`.

```json
{
  "walletAddress": "0x…",
  "verifierIds": ["proof-of-human"],
  "requireAll": true
}
```

## Output

- **`already_verified`** — policy satisfied.
- **`verification_required`** — open **`hostedVerifyUrl`** (and use **`requiredVerifiers`** as needed).

Schemas are loaded from the server; no need to memorize every field.
