Use `neus_verifiers_catalog` at the start of an MCP workflow when the client needs the live verifier catalog.

## Best For

- tool-driven discovery
- verifier selection
- avoiding hardcoded verifier metadata

## Upstream API

`GET /api/v1/verification/verifiers`

## Input

```json
{}
```

## Stable Output

- verifier list
- verifier labels and types
- field metadata needed for policy or UI
