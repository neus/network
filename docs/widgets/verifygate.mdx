`VerifyGate` checks eligibility, runs **Hosted Verify** when a new proof is needed, then renders children. Pass **`appId`** and **`billingWallet`** from Hub so checks bill your account. Visibility: [Security and trust](../platform/security-and-trust). Setup: [Register your app](../platform/setup).

## Default create behavior

When **`proofOptions`** is omitted, create mode merges:

| Field | Default |
|-------|---------|
| `privacyLevel` | `private` |
| `publicDisplay` | `false` |

Pass **`proofOptions`** only when you need **unlisted public** or **listed public**.

## Basic usage

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

export function Page() {
  return (
    <VerifyGate
      requiredVerifiers={['nft-ownership']}
      verifierData={{
        'nft-ownership': {
          contractAddress: '0x...',
          tokenId: '1',
          chainId: 8453
        }
      }}
      appId="your-app-id"
      billingWallet="0xYourHubWallet"
    >
      <div>Unlocked</div>
    </VerifyGate>
  );
}
```

## Key props

| Prop | What it does |
|------|--------------|
| `requiredVerifiers` | Sets the verification policy |
| `verifierData` | Supplies verifier-specific inputs |
| `strategy` | Chooses reuse, reuse-or-create, or fresh |
| `proofOptions` | Overrides privacy and content storage (defaults are private) |
| `mode` | Chooses create vs access behavior |
| `qHash` | Existing id for access mode |
| `appId` | Public product ID from Hub |
| `billingWallet` | Your Hub wallet â€” checks bill your credits, not the end user |
| `hostedCheckoutUrl` | Overrides the hosted verify URL when needed |
| `oauthProvider` | Pre-selects the OAuth provider when the user already chose one in your app |
| `wallet` | Explicit provider for direct signing. Use this for non-EVM wallets and pass CAIP-2 `chain` context. |

## Typical uses

- Paid or premium content behind a check
- NFT or token access
- Hosted social or organization verification
- Fresh proof for high-stakes actions

## Hosted OAuth

Interactive verifiers (**`ownership-social`**, **`ownership-org-oauth`**, **`proof-of-human`**) use **[Hosted Verify](../cookbook/auth-hosted-verify)**. Set `oauthProvider` when the user already picked a provider in your app.

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

export function ConnectGithub() {
  return (
    <VerifyGate
      requiredVerifiers={['ownership-social']}
      oauthProvider="github"
      appId="your-app-id"
      billingWallet="0xYourHubWallet"
    />
  );
}
```

Allowed values:

- **Social:** `x`, `twitter`, `github`, `discord`, `facebook`, `linkedin`, `telegram`, `farcaster`
- **Org:** `google`, `microsoft`

**Popup:** still **`neus_checkout_done`**. `oauthProvider` only picks which sign-in opens first; proof payload and your handling stay the same.

## Direct-sign vs hosted

- **`ownership-basic`** and other wallet-creatable verifiers can run in-widget with a connected EVM wallet. For non-EVM direct signing, pass `wallet` and CAIP-2 `chain` explicitly.
- Social, org, and human flows route through hosted verify. Start from **[Hosted Verify](../cookbook/auth-hosted-verify)** when you are unsure.
