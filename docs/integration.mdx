Check before you ask users to verify. Save receipt IDs. Reuse everywhere.

**Before you integrate:** **[Get started](./get-started)** (platform, app link, credits, sponsor). **Paths:** **[Choose an integration path](./choose-an-integration-path)**. **Billing:** **[Billing](./platform/billing)**.

## Loop

1. **Check** — Already verified?
2. **Verify** — Only if needed (usually [Hosted Verify](./cookbook/auth-hosted-verify)).
3. **Save** — Proof receipt ID with the user record.
4. **Reuse** — Display, gates, policy, agents.

## Checklist

<Checklist>
  <Item>Pick Hosted, Widgets, SDK, API, or MCP (one primary path)</Item>
  <Item>Gate before prompting for verification</Item>
  <Item>Persist proof IDs, not raw signatures</Item>
  <Item>Test: verify → save → check → reuse</Item>
  <Item>Complete [Get started](./get-started) for platform setup and payer model; use [Billing](./platform/billing) for edge cases</Item>
</Checklist>

## Verification strategies (SDK / API)

| Strategy | When |
|----------|------|
| `reuse-or-create` | Default UX |
| `fresh` | High-stakes actions |
| `reuse` | Read-only, never create |
