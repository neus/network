Same loop on every surface: **Check, verify, save, reuse**.

- **Surfaces:** [Choose an integration path](./choose-an-integration-path) - Hosted Verify, SDK, widgets, API, MCP.
- **Platform:** [Billing](./platform/billing), [Verification results](./platform/receipts-and-results), [Security and trust](./platform/security-and-trust), [LLM docs](./platform/llm-docs)

**Check** whether the person or wallet already has a valid result (saved **verification ID** or a server-side eligibility check) before asking them to verify again. **Save** the **verification ID** from the response ([Verification results](./platform/receipts-and-results)). The API field is **`proofId`**.

## Loop

1. **Check** - saved id or proofs/check
2. **Verify** - only if needed (hosted, SDK, or API)
3. **Save** - verification ID on user/session
4. **Reuse** - UI, VerifyGate, server, MCP

| Situation | Approach |
|-----------|----------|
| Repeat visits | Run an eligibility check or use a stored verification ID before a new verify |
| Balance / NFT gates | Narrow `since` or `fresh` strategy - [SDK verifications](./sdks/verifications) |
| Org and domain | [Platform setup](./platform/setup), then the same loop |

## Visibility

[Security and trust](./platform/security-and-trust) has the full matrix.

| Mode | Use |
|------|-----|
| Listed public | Discoverable attribution |
| Unlisted public | Gates and reuse, not listed |
| Owner-only | Signed-in owner |

SDK `client.verify()` defaults **private**. VerifyGate create defaults **unlisted public**.

## Strategies (SDK / API)

| Strategy | When |
|----------|------|
| `reuse-or-create` | Default: reuse if valid, else verify |
| `fresh` | Always new verification |
| `reuse` | Existing verification only |

More filters: [SDK verifications](./sdks/verifications).
