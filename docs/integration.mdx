Build the canonical loop: verify once, save the receipt, and reuse it across gates, actions, and agents.

**Before you integrate:** start with [Get started](./get-started) for app linking, credits, and sponsor setup. Then choose [Hosted Verify](./cookbook/auth-hosted-verify), [Widgets](./widgets/overview), [SDK](./sdks/overview), [API](./api/overview), or [MCP](./mcp/overview).

## Loop

1. **Check** - Is the user already verified?
2. **Verify** - Only if needed, usually with [Hosted Verify](./cookbook/auth-hosted-verify).
3. **Save** - Store the proof receipt ID with the user record.
4. **Reuse** - Display it, gate it, attach it, or audit it.

## Checklist

<Checklist>
  <Item>Pick one primary path: Hosted Verify, Widgets, SDK, API, or MCP</Item>
  <Item>Gate before prompting for verification</Item>
  <Item>Persist proof IDs, not raw signatures</Item>
  <Item>Test verify -> save -> check -> reuse</Item>
  <Item>Complete [Get started](./get-started) before production volume</Item>
</Checklist>

## Verification strategies

| Strategy | When |
|----------|------|
| `reuse-or-create` | Default user experience |
| `fresh` | High-stakes actions |
| `reuse` | Read-only checks |
