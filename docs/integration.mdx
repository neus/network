Check eligibility first. Verify only when needed. Reuse everywhere.

## The Loop

```

1. **Check** — Is the user already verified?
2. **Verify** — Only if no valid proof exists
3. **Save** — Store the proof receipt ID
4. **Reuse** — Same proof for all downstream needs

---

## Production Checklist

<Checklist>
  <Item>Pick your integration path (Hosted, SDK, Widgets, API, or MCP)</Item>
  <Item>Implement gate checks before prompting for verification</Item>
  <Item>Use hosted verify for interactive flows</Item>
  <Item>Save proof receipt IDs with the records that need them</Item>
  <Item>Reuse the same proof for display, access, and policy</Item>
  <Item>Test the full loop: verify → save → check → reuse</Item>
</Checklist>

---

## Verification Strategies

| Strategy | Use When | Behavior |
|----------|----------|----------|
| `reuse-or-create` | Standard UX | Reuse existing proof, create if missing |
| `fresh` | High-stakes actions | Always create a new proof |
| `reuse` | Read-only checks | Never create new proofs |

## Go-Live Rules

1. **Single hosted path** — Use `/verify` for all interactive flows
2. **Live catalog** — Read verifier metadata from the API
3. **Persist IDs** — Save proof receipt IDs, not raw signatures
4. **Freshness windows** — Use time bounds for point-in-time checks
5. **Full loop testing** — Verify → save → display → check → reuse
