## Loop

| Step | Do this |
|------|---------|
| 1 | Check a saved **`qHash`** or run eligibility (`gateCheck`, proofs check) |
| 2 | Run verify **only** if missing or you need a fresh receipt |
| 3 | Persist **`qHash`** on your user or record |
| 4 | Reuse on return visits and for access rules |

## Where code runs

| Layer | Use |
|-------|-----|
| Browser | `VerifyGate`, `getHostedCheckoutUrl` → `https://neus.network/verify`, SDK polling |
| Server | `NEUS_ACCESS_KEY`, `verifyFromApp`, `gateCheck`, raw HTTP behind your API |

Passkey, wallet, and OAuth for **default** flows: **NEUS hosted page**, not your UI.

Use the SDK from the browser. Do not ship **`npk_*`** or credentialed `fetch` to `api.neus.network` from client bundles; proxy through your backend if you need custom headers. [JavaScript SDK](./sdks/javascript) · [API overview](./api/overview).

Identity and chain terms are centralized in [CAIP-380 Portable Proof](./learn/standards/caip-380): CAIP-2 is network context, CAIP-10 is account-id shape, and `did:pkh` is the stable DID representation.

**MCP and agent wallets** (not the same as app `VerifyGate` flows): [MCP overview](./mcp/overview) and [Agent verification flow](./agents/agent-verification-flow).
