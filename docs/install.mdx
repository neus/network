Connect once. Your assistant carries who you are and what it may do across editors. It reuses saved results when it can, then reports **Passed**, **Action needed**, or **Blocked**.

## Install

```bash
npm i -g @neus/sdk
neus setup
neus check
```

No global install? Run `npx -y -p @neus/sdk neus setup` once.

Ask your assistant:

```txt
Use NEUS for my identity and what I allow. Reuse saved results when you can.
```

Example prompts: [MCP journeys](./mcp/journeys).

## Supported tools

- Cursor
- Claude Code
- Codex
- VS Code
- MCP-compatible clients

## Bring your own agent

Already have agents in Cursor, Claude Code, or Claude Desktop? NEUS does not replace them. It adds identity, authority, and saved results on top.

**Import existing IDE config:**

```bash
neus import --from auto
```

Detects Cursor, Claude Code, and Claude Desktop workspaces and snapshots their config into a portable NEUS manifest. Use `--from cursor`, `--from claude-code`, or `--from claude-desktop` to target one.

**Connect a registered agent to a project:**

```bash
neus mount <agentId> --apply cursor
```

Writes `.neus/mount.json` with the agent's verified identity, permissions, and host rules. Run it again after permission changes. See [Connect agent context](./agents/runtime-mount).

**Agent frameworks:** Any agent that can sign with a wallet or complete a hosted step works the same way: identity for who they are, authority for what they may do. Results travel across runtimes. See [Agent verification flow](./agents/agent-verification-flow).
