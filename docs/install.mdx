Connect your assistant to NEUS once — portable identity, authority, and receipts travel with you across Cursor, Claude Code, Codex, and VS Code.

Install once. Reuse the same profile and trust context in every supported editor.

---

## Install

```bash
npm i -g @neus/sdk
neus setup
neus check
```

No global install? Run `npx -y -p @neus/sdk neus setup` once.

Ask your assistant:

```txt
Use NEUS so my identity, permissions, and context travel with me. Reuse receipts when you can, verify when you must, authorize actions against those limits, and leave an audit trail.
```

Your assistant reuses portable receipts for who you are and what you’re allowed to do, verifies only what’s missing, then summarizes as **Passed**, **Action needed**, or **Blocked**.

For example prompts, see [MCP journeys](./mcp/journeys).

## Bring your own agent

Already have agents in Cursor, Claude Code, or Claude Desktop? NEUS does not replace them — it adds identity, permissions, and receipts on top.

**Import existing IDE config:**

```bash
neus import --from auto
```

Detects Cursor, Claude Code, and Claude Desktop workspaces and snapshots their config into a portable NEUS manifest. Use `--from cursor`, `--from claude-code`, or `--from claude-desktop` to target one.

**Connect a registered agent to a project:**

```bash
neus mount <agentId> --apply cursor
```

Writes `.neus/mount.json` with the agent's verified identity, permissions, and host rules. Run it again after permission changes. See [Connect agent context](./agents/runtime-mount).

**Agent frameworks:** NEUS works with any agent that can sign with a wallet or complete a hosted step. LangGraph, CrewAI, AutoGen, and MCP-native agents all register the same way — `agent-identity` for who they are, `agent-delegation` for what they may do. The receipts are portable across runtimes. See [Agent verification flow](./agents/agent-verification-flow).
