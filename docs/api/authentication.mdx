**Default:** ceremony on **`neus.network/verify`** or on the server. **Exception:** you explicitly sign inside your own page ([Signing format](../verification/signing-format)).

| Need | Mechanism | Wallet code in your app |
|------|-----------|---------------------------|
| MCP client auth | OAuth 2.0 + PKCE | No |
| Receipts with your existing login | App Link + `verifyFromApp` | No |
| Browser ceremony | Hosted Verify / `VerifyGate` | No |
| Keys / raw HTTP | Bearer `npk_*` | No |
| Sign inside your page | SDK `verify` + `wallet` | Yes — rare |

## MCP OAuth

MCP clients (Claude Code, OpenAI, VS Code, Cursor) authenticate via **OAuth 2.0 Authorization Code with PKCE**. The flow wraps the existing NEUS hosted login:

```text
/oauth/authorize (validates OAuth params, renders HostedLoginFlow)
→ user signs in on NEUS
→ backend issues one-time auth code
→ client exchanges code for MCP access token
→ MCP requests use Bearer token
```

The MCP access token is a JWT with `aud=https://mcp.neus.network/mcp` and `token_use=mcp_access`. It cannot be used for general API access.

See [MCP OAuth](../mcp/oauth) for the full reference. Quick setup:

```bash
npx -y -p @neus/sdk neus setup
npx -y -p @neus/sdk neus auth
```

## App Link

`Authorization: Bearer <npk_...>`, `x-neus-app`, linked user address, active App Link. [Quickstart](../quickstart).

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: process.env.NEUS_APP_ID,
  apiKey: process.env.NEUS_ACCESS_KEY,
});

const receipt = await client.verifyFromApp({
  user: { walletAddress: user.walletAddress },
  verifier: 'ownership-basic',
  content: { claim: 'User completed onboarding', source: 'your-app' },
});
```
