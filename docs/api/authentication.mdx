**Default:** [Hosted Verify](../cookbook/auth-hosted-verify) or the **SDK** — users never need raw HTTP signing details.

Use the paths below only when you intentionally call the HTTP API without the SDK.

## Wallet signing standards

| Type | Standard | Typical wallets |
|------|----------|-----------------|
| EOA | EIP-191 | MetaMask, Coinbase, Rainbow |
| Smart contract | EIP-1271 | Safe, Argent, Sequence |
| Counterfactual | EIP-6492 | Undeployed smart accounts |

## Programmatic proof creation (HTTP)

1. **`POST /api/v1/verification/standardize`** — request body for your verifiers; response includes **`signerString`**.
2. Sign **`signerString`** with the user’s wallet (same request fields you will submit).
3. **`POST /api/v1/verification`** — same fields as step 1, plus **`signature`**.

Request and response shapes: **OpenAPI** (`docs/openapi/public-api.json`). Narrative: [Signing format](../verification/signing-format).

## Embedded / picky wallets

Use **`signMessage`** from **`@neus/sdk`** instead of calling the wallet with a hand-built payload.
