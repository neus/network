NEUS does not replace your login system. Most apps keep their own login and use NEUS for trust receipts.

## Choose a path

| Need | Use | What users see |
| --- | --- | --- |
| Keep your existing login | **App Link** + server SDK | No wallet popup for each request |
| Let NEUS host the account step | **Hosted Verify** | Passkey-first sign-in, wallet when needed |
| Gate a React surface | **VerifyGate** | Inline check, hosted handoff only when needed |
| Direct wallet proof in your UI | **SDK verify** | Wallet signature in your app |
| Backend reads and checks | **Personal access key** | No user-facing flow |

Use raw HTTP signing only without the SDK.

## App Link

This is the default server path. Your backend calls NEUS with your app ID and access key, and NEUS checks the user's App Link before creating the receipt.

Requirements:

- `Authorization: Bearer <npk_...>` from [Access keys](https://neus.network/profile?tab=account)
- `x-neus-app` matching the linked app
- A user wallet or account address in the request body
- An active App Link for that user and app

The SDK handles this with `verifyFromApp`:

```javascript
const client = new NeusClient({
  appId: process.env.NEUS_APP_ID,
  apiKey: process.env.NEUS_ACCESS_KEY,
});

const receipt = await client.verifyFromApp({
  user: { walletAddress: user.walletAddress },
  verifier: 'ownership-basic',
  content: 'User completed onboarding',
});
```

See [Quickstart](../quickstart) for the full flow.

## Hosted Login and Verify

Hosted Verify is the NEUS browser flow at `https://neus.network/verify`. Use it when the user needs to sign in on NEUS, complete OAuth, pass a human check, connect a wallet, or finish a gate.

The hosted login path is passkey-first.

```javascript
const url = getHostedCheckoutUrl({
  verifiers: ['ownership-basic'],
  returnUrl: 'https://yourapp.com/neus/callback',
});
```

After completion, store the returned `qHash` with your own user or record. See [Hosted Verify](../cookbook/auth-hosted-verify).

## Personal Access Keys

Use **`Authorization: Bearer`** with a key from **[Access keys](https://neus.network/profile?tab=account)** for server-side API calls.

Keep access keys on servers only. Do not put `npk_*` keys in browser bundles.

## Direct Signing

Use SDK signing when the user is already in your app and you want a wallet signature there:

```javascript
await client.verify({
  verifier: 'ownership-basic',
  content: 'Hello NEUS',
  wallet: window.ethereum,
});
```


## Raw HTTP Signing

Only without Hosted Verify or the SDK:

1. Build the JSON body.
2. `POST /api/v1/verification/standardize`.
3. Sign the returned `signerString`.
4. `POST /api/v1/verification` with the same body and `signature`.

Use `signMessage` from `@neus/sdk` when possible. Details: [Signing format](../verification/signing-format).
