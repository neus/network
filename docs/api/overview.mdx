## Quick Reference

| Surface | URL |
|---------|-----|
| Base URL | `https://api.neus.network` |
| OpenAPI Spec | `/openapi/public-api.json` |
| Verifier Catalog | `GET /api/v1/verification/verifiers` |

## Authentication

Most read endpoints are public. Write operations require request-bound wallet signatures.

<Note>
For manual signing, use `POST /api/v1/verification/standardize` to get the exact signing string.
</Note>

## Core Endpoints

| Operation | Method | Path |
|-----------|--------|------|
| Create verification | POST | `/api/v1/verification` |
| Standardize signing string | POST | `/api/v1/verification/standardize` |
| Poll status | GET | `/api/v1/verification/status/{qHash}` |
| Gate check | GET | `/api/v1/proofs/check` |
| Revoke proof | POST | `/api/v1/proofs/revoke-self/{qHash}` |
| Verifier catalog | GET | `/api/v1/verification/verifiers` |

## Proof Receipt Naming

Treat the proof receipt ID as opaque and stable.

<Note>
`qHash` is the HTTP and path name for the same proof receipt ID used elsewhere as `proofId`.
</Note>

## Create Verification

```bash
POST /api/v1/verification
```

Submit a verification request after you have either collected the standard signing string or completed a hosted flow.

## Standardize Signing String

```bash
POST /api/v1/verification/standardize
```

Use this endpoint when you need the exact deterministic wallet message that must be signed.

## Poll Status

```bash
GET /api/v1/verification/status/{qHash}
```

Pass the proof receipt ID as the `qHash` path parameter.

## Revoke Proof

```bash
POST /api/v1/proofs/revoke-self/{qHash}
```

Pass the proof receipt ID as the `qHash` path parameter.

## Gate Check

```bash
GET /api/v1/proofs/check?address=0x...&verifierIds=ownership-basic
```

Use this for server-side policy enforcement when you only need eligibility.

## Verifier Catalog

```bash
GET /api/v1/verification/verifiers
```

Read the live catalog instead of hardcoding verifier labels, input requirements, or hosted-flow assumptions.
