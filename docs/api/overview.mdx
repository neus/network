Use the SDK when you can; use HTTP when your backend or automation needs direct control.

**Base:** `https://api.neus.network`

**Server integration?** Complete [Get started](../get-started) first for app attribution, sponsor grants, and billing setup.

## Endpoints

| Path | Method | Purpose |
|------|--------|---------|
| `/api/v1/verification` | POST | Create proof receipt |
| `/api/v1/verification/standardize` | POST | Build the signing string |
| `/api/v1/verification/status/{qHash}` | GET | Fetch proof status |
| `/api/v1/proofs/check` | GET | Check eligibility |
| `/api/v1/proofs/by-wallet/{address}` | GET | List proofs for a wallet |
| `/api/v1/verification/verifiers` | GET | List verifiers |

## Examples

**Proofs by wallet**

```bash
curl "https://api.neus.network/api/v1/proofs/by-wallet/0x...?limit=50"
```

Public and unlisted proofs are visible anonymously; private proofs need the user signed in.

**Gate check**

```bash
curl "https://api.neus.network/api/v1/proofs/check?address=0x...&verifierIds=ownership-basic"
```

**Create proof**

```text
standardize -> sign -> POST /verification
```

**Verifier catalog**

```bash
curl https://api.neus.network/api/v1/verification/verifiers
```

## Auth

- Reads: public and unlisted without login; private when the user is signed in.
- Writes: wallet signature or session auth. See [Authentication](./authentication).
