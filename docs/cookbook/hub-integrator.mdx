Use this guide when you are connecting a production app to NEUS hosted verify, app attribution, and sponsored verification flows.

## Prerequisites

- A NEUS account with a verified wallet
- Your public app domain
- If using hosted auth code exchange, partner access enabled during onboarding

## 1. Domain Verification

Link your domain to your wallet by setting a DNS TXT record.

**Step 1:** Add a DNS TXT record at `_neus.<yourdomain.com>`:

```text
neus=<your-wallet-address>
```

**Step 2:** Create a domain proof:

```javascript
const proof = await client.verify({
  verifier: 'ownership-dns-txt',
  data: {
    domain: 'yourdomain.com',
    walletAddress: '0x...',
  },
  walletAddress: '0x...',
  signature: '0x...',
  signedTimestamp: Date.now(),
});
```

**Step 3:** Confirm the proof is active:

```bash
GET /api/v1/proofs/check?verifierIds=ownership-dns-txt&address=0x...
```

## 2. Organization Membership Verification

For `ownership-org-oauth`, the public integration path is simple: send the user to hosted NEUS verify and let NEUS handle Google or Microsoft.

### What You Actually Need To Do

1. Launch `https://neus.network/verify`
2. Use `preset=org-pack` when organization membership is the only requirement
3. Add `returnUrl` or popup mode for completion handling
4. Read the returned proof receipt ID after the hosted flow finishes

### Recommended Hosted URL

```text
https://neus.network/verify?preset=org-pack&returnUrl=https://app.example.com/verified
```

Popup example:

```text
https://neus.network/verify?preset=org-pack&mode=popup&origin=https://app.example.com
```

### Restrict to One Company Domain Only When You Need It

If access should only succeed for members of one company, pass `expectedOrgDomain`.

```jsx
<VerifyGate
  hostedCheckoutUrl="https://neus.network/verify"
  requiredVerifiers={['ownership-org-oauth']}
  verifierData={{
    'ownership-org-oauth': {
      provider: 'google',
      expectedOrgDomain: 'acme.com',
    },
  }}
>
  <ProtectedContent />
</VerifyGate>
```

<Note>
Most integrators do not need to create their own Google or Microsoft OAuth app, manage redirect URIs, or register provider credentials just to use the hosted NEUS flow. If NEUS coordinates a custom enterprise provider setup, treat that as a private onboarding task, not the default public integration path.
</Note>

## 3. Link App (Hub UI)

Link your integration domain in the Hub to receive credits and attribution:

| Step | Action |
|------|--------|
| 1 | Go to [Hub](https://neus.network/hub) -> Profile -> Apps |
| 2 | Enter app domain, for example `app.example.com` or `https://app.example.com` |
| 3 | Set daily credit cap |
| 4 | Use `*` for any origin only in development |
| 5 | Send `X-Neus-App` and `Origin` headers from your approved domain |

## 4. App Attribution (API)

To track usage per integration via API:

1. Create or obtain an `appId` for your integration
2. Treat `appId` as a public attribution identifier, not a secret
3. Include `X-Neus-App: <appId>` on all API requests

```javascript
const client = new NeusClient({ appId: 'your-app-id' });
```

## 5. Verify with NEUS (Hub UI)

Verify domains and organizations directly in the Hub:

| Flow | Where | Notes |
|------|-------|------|
| Domain | Profile -> Identity -> Domains | Add DNS TXT at `_neus.<domain>` first |
| Organization | Profile -> Identity -> Organizations | Hosted NEUS verify for Google Workspace or Microsoft 365 |
| Privacy | Per proof | Public or private proof visibility |

Link your app first in Profile -> Apps, then verify your domain or organization.

## 6. Hosted Checkout for Interactive Verifiers

Interactive verifiers (`ownership-social`, `ownership-org-oauth`, `proof-of-human`) should resolve through the NEUS-hosted flow.

### React (VerifyGate)

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

<VerifyGate
  apiUrl="https://api.neus.network"
  hostedCheckoutUrl="https://neus.network/verify"
  requiredVerifiers={['ownership-org-oauth']}
  verifierData={{
    'ownership-org-oauth': {
      provider: 'google',
      expectedOrgDomain: 'acme.com',
    },
  }}
  onVerified={(result) => console.log('Verified:', result.proofId)}
>
  <ProtectedContent />
</VerifyGate>
```

If you do not need to restrict the company domain, prefer the simpler preset URL and avoid custom verifier payloads.

## 7. Sponsor Grants

If your integration sponsors verification costs:

```javascript
const client = new NeusClient({
  appId: 'your-app-id',
  sponsorGrant: '<sponsor-grant-token>',
});
```

<Warning>
Sponsor grant tokens are short-lived. Re-issue them per session. Do not embed long-lived API keys in client-side code.
</Warning>

## 8. Usage and Analytics

Track usage with your `appId` and NEUS partner reporting:

- Proof creation volume per verifier
- Gate check counts by app
- Sponsor grant consumption

The `/api/v1/proofs/check` response includes `matchCount` for your telemetry.

## Operational Rules

- Keep `/verify` as the single hosted path for interactive verification
- Route all OAuth and interactive verifier handoff through NEUS-hosted surfaces
- Do not create parallel auth routes when NEUS already provides a standard path
