**Default.** The user finishes on NEUS. Your app stores the **proof ID**. **Check** before asking again. [Integration](../integration#loop). **Billing.** [Billing](../platform/billing).

Build links with the SDK (`returnUrl`). Read the result with the SDK after return.

**Automation.** [MCP](../mcp/overview). Interactive steps use this hosted URL.

## When

| Need | Why hosted |
|------|------------|
| Login | Guided signer |
| Social | OAuth in NEUS |
| Human / org | Interactive |

## URL

Base: `https://neus.network/verify`

| Param | Role |
|-------|------|
| `intent=login` | Session-first |
| `verifiers` | Comma-separated ids |
| `preset` | Bundled set |
| `gateId` | Gate requirements |
| `returnUrl` | Next page |
| `mode=popup` | Popup + `postMessage` |
| `origin` | Popup opener |

## Auth surfaces

Passkeys require a **secure context** (HTTPS or localhost). Browsers may offer cross-device or security-key flows.

| Surface | Passkeys | Wallet |
|--------|----------|--------|
| Desktop | Device passkeys, handoff, cross-device | Connect from page; QR needs a gesture |
| Mobile / PWA | OS or browser passkey UI when available | Prefer wallet app / deep link |

Use **https** or **http://localhost** with an origin that matches the app domain you configured for Hosted Verify—not raw LAN IPs.

PWAs: do not cache Hosted Verify or login responses in your service worker.

`getHostedCheckoutUrl` builds a **Hosted Verify** URL - the same `https://neus.network/verify` flow as the query-string examples below.

## Code

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

window.location.href = getHostedCheckoutUrl({
  verifiers: ['ownership-basic'],
  returnUrl: 'https://myapp.com/callback',
});
```

**Popup:** `mode: 'popup'`, `origin: window.location.origin`, `window.open`, listen for `neus_checkout_done` / `neus_login_complete`. **Redirect:** handle `returnUrl` + SDK.
