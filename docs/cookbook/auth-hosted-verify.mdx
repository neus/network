**Shipping an app?** Start with **[Get started](../get-started)** for app linking, credits, and sponsor setup. Then use Hosted Verify in your flow.

Users complete verification on `neus.network` with passkey or wallet. Your app supplies `returnUrl` and builds the link with the SDK.

After they finish, complete the session in your app with `@neus/sdk` and use `Authorization: Bearer` on API calls.

For MCP, prefer tools that return `neus.network/verify` links. Keep the browser handoff simple and let NEUS handle the verification UI.

## When to use

| Need | Why hosted |
|------|------------|
| Login or signup | Guided signer |
| Social linking | OAuth in NEUS |
| Human or org checks | Interactive flows |

## URL

**Base:** `https://neus.network/verify`

| Param | Role |
|-------|------|
| `intent=login` | Session-first login |
| `verifiers` | Comma-separated IDs |
| `preset` | Bundled verifier set |
| `gateId` | Gate requirements |
| `returnUrl` | Where to send the user next |
| `mode=popup` | Popup + `postMessage` |
| `origin` | Opener origin for popup flows |

## Code

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

const verifyUrl = getHostedCheckoutUrl({
  verifiers: ['ownership-basic'],
  returnUrl: 'https://myapp.com/callback',
});

window.location.href = verifyUrl;
```
