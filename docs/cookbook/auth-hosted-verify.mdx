Hosted Verify is the browser flow at `https://neus.network/verify`. Use it when you want NEUS to handle the user-facing step instead of building verification UI yourself.

## What it handles

| Need | Hosted behavior |
| --- | --- |
| Login | Passkey-first NEUS session, wallet supported when needed |
| Social or org checks | OAuth completes on NEUS |
| Human checks | User completes the provider flow on NEUS |
| Wallet-required proofs | User connects and signs only when that proof needs it |
| Gate access | User completes requirements, then returns to your app |

Hosted Verify is the handoff for trust receipts, provider steps, and gated access.

## URL

```txt
https://neus.network/verify
```

Common parameters:

| Param | Use |
| --- | --- |
| `intent=login` | Open hosted login |
| `verifiers` | Comma-separated verifier IDs |
| `preset` | Bundled verifier set |
| `gateId` | Gate requirements |
| `returnUrl` | Redirect target after completion |
| `mode=popup` | Popup handoff |
| `origin` | Expected opener origin for popup messages |

## Redirect

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

window.location.href = getHostedCheckoutUrl({
  verifiers: ['ownership-basic'],
  returnUrl: 'https://myapp.com/neus/callback',
});
```

After completion, read the returned `qHash`, save it with your user or record, and check it before asking the user to verify again.

## Popup

For embedded flows, open Hosted Verify with `mode: 'popup'` and `origin: window.location.origin`. Listen for:

- `neus_checkout_done`
- `neus_login_complete`

The popup response includes the receipt result when verification completes.

## When to use App Link instead

If your backend already has the user and you want no browser handoff, use [App Link](../quickstart) with `verifyFromApp`.
