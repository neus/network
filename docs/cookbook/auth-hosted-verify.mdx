**Shipping an app?** **[Get started](../get-started)** for platform **`appId`**, credits, and sponsor setup; then use hosted verify in your flow.

Users complete verification on **neus.network** (passkey or wallet). Your app supplies **`returnUrl`** and builds the link with the SDK — you do not need custom wallet UI.

After they finish, **complete the session in your app with `@neus/sdk`** and use **`Authorization: Bearer`** on API calls — [SDK Authentication](../sdks/authentication).

**MCP:** Prefer tools that return **`neus.network/verify`** links; optional **Bearer** after your user is signed in — [MCP Auth](../mcp/auth).

## When to use

| Need | Why hosted |
|------|------------|
| Login / signup | Guided signer |
| Social linking | OAuth in NEUS |
| Human / org checks | Interactive flows |

## URL

**Base:** `https://neus.network/verify`

| Param | Role |
|-------|------|
| `intent=login` | Session-first login |
| `verifiers` | Comma-separated IDs |
| `preset` | Bundled verifier set |
| `gateId` | Gate requirements |
| `returnUrl` | Where to send the user next |
| `mode=popup` | Popup + `postMessage` |
| `origin` | Opener origin (popup) |

## Auth surfaces

How hosted login behaves by environment (passkeys use **WebAuthn** in a **secure context**; a missing *platform* authenticator does not block **cross-device** or **security-key** prompts).

| Surface | Passkeys | Wallet (AppKit / WalletConnect) |
|--------|----------|----------------------------------|
| Desktop browser or IDE embedded browser | May complete locally or via phone / cross-device handoff when the browser offers it | Use **Connect Wallet** from the page; QR and mobile scan rely on a normal user gesture so the wallet modal and popups are not suppressed |
| Mobile browser or installed PWA | Uses the platform password manager or authenticator when available | Prefer the wallet app or deep link the browser offers |
| Local development | Use **`http://localhost:2800`** (hostname `localhost`) so **rpID** and origin match; avoid raw LAN IPs for WebAuthn | Same as desktop; keep the app origin aligned with your NEUS passkey profile |

**PWA:** The installable shell uses a minimal service worker that **does not intercept** `/api/` routes, so session and auth traffic stay on the network stack. Do not add auth-specific caching in the service worker for hosted flows.

## Code

```javascript
import { getHostedCheckoutUrl } from '@neus/sdk';

const verifyUrl = getHostedCheckoutUrl({
  verifiers: ['ownership-basic'],
  returnUrl: 'https://myapp.com/callback',
});

window.location.href = verifyUrl;
// Popup: add mode: 'popup', origin: window.location.origin, then window.open(...)
```

**Popup:** listen for **`neus_checkout_done`** (includes `proofId`) or **`neus_login_complete`**. **Redirect:** handle your **`returnUrl`** route and finish login with the SDK.
