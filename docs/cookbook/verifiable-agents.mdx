`agent-identity` (agent wallet signs) and `agent-delegation` (approving account signs) are **separate** proofs. One person can do both, or roles can split.

For apps creating trust receipts server-side for existing users, use **App Link** (`agent-delegation` with `scope: 'app-link'`). See the [Quickstart](../quickstart). [Agent concepts](../agents/concepts)

## MCP

<Steps>
 <Step title="Check">
 Call **`neus_agent_link`** with the **`agentWallet`** after **`neus_context`** (and **`neus_me`** when Bearer is set). Stop if **`linked: true`**.
 </Step>
 <Step title="Prepare (only if missing)">
 Call **`neus_agent_create`** when you need signing material or a prepared hosted URL for identity + delegation.
 </Step>
 <Step title="Finish">
 Open **`hostedVerifyUrl`** and/or submit **`neus_verify`** using the payloads from the create response.
 </Step>
 <Step title="Confirm">
 Call **`neus_agent_link`** again until **`linked: true`**.
 </Step>
</Steps>
