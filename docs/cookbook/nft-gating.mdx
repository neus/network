Verifier: [`nft-ownership`](../verification/nft-ownership). Ownership is **point-in-time** - enforce `since` / fresh proofs for high-stakes gates.

## Widget

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

export function NFTGatedContent() {
  return (
    <VerifyGate
      requiredVerifiers={['nft-ownership']}
      verifierData={{
        'nft-ownership': {
          contractAddress: '0x...',
          tokenId: '1',
          chainId: 8453,
        },
      }}
      appId="my-nft-gate"
    >
      <div>Exclusive content</div>
    </VerifyGate>
  );
}
```

## Server: gateCheck

```javascript
const result = await client.gateCheck({
  address: '0x...',
  verifierIds: ['nft-ownership'],
  contractAddress: '0x...',
  tokenId: '1',
  chainId: 8453,
  since: Date.now() - 60 * 60 * 1000,
});

if (!result.data?.eligible) {
  throw new Error('Access denied');
}
```

## Multiple token configs

```jsx
const allowedTokens = [
  { contractAddress: '0x...', tokenId: '1', chainId: 8453 },
  { contractAddress: '0x...', tokenId: '2', chainId: 8453 },
];

export function NFTGatedContent({ selectedToken }) {
  return (
    <VerifyGate
      requiredVerifiers={['nft-ownership']}
      verifierData={{ 'nft-ownership': allowedTokens[selectedToken] }}
      appId="my-nft-gate"
    >
      <div>Holders of the selected NFT</div>
    </VerifyGate>
  );
}
```

`VerifyGate` accepts one `nft-ownership` object per check. For “any of these tokens,” let the user choose the token to verify, or perform multiple server-side `gateCheck` calls and allow access when one is eligible.
