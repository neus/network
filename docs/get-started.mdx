## App verification

1. Sign in at [neus.network](https://neus.network) and open **Gates** in your profile.
2. Create a gate, choose the checks visitors must pass, and **Publish**.
3. Copy the **checkout link** or **`<VerifyGate gateId="..." />`** embed from the publish step.
4. Optionally run **`gateCheck({ gateId, address })`** on your server before trusted actions.

Users complete checks on NEUS. They do not need NEUS accounts or API keys. The published gate owns the checkout policy, verifier set, and default billing.

## Agent trust

1. `npx -y -p @neus/sdk neus import` (optional)
2. **`neus_context`** → **`neus_agent_link`**
3. **`neus_agent_create`** when proofs are missing — finish on NEUS
4. **`neus_agent_link`** until **`linked: true`**

Details: [Agent identity](./agents/agent-identity) · [Agent delegation](./agents/agent-delegation)

## Assistant setup

**CLI-supported hosts:** Cursor, VS Code/Codex, and Claude Code.

```bash
npx -y -p @neus/sdk neus setup
npx -y -p @neus/sdk neus auth
npx -y -p @neus/sdk neus doctor --live
```

Use **`neus auth`** for OAuth in your browser. Access keys are for servers, CI, and automation — never in browser code. Call **`neus_context`** first in each MCP session.
