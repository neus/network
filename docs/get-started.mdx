**Most teams start here:** [Hosted Verify](./cookbook/auth-hosted-verify) → save **proof receipt ID** → reuse with checks ([Integration](./integration#loop)). **For most integrations, your app pays:** org credits + **sponsor grants**; end users are not expected to manage billing first ([Choose who pays](#choose-who-pays) below).
</Note>

<Tip>
**Loop:** [Check](./integration#loop) → verify if needed → save **proof receipt ID** → reuse. Interactive flows: **[Hosted Verify](./cookbook/auth-hosted-verify)**. [Proof receipts](./platform/receipts-and-results)
</Tip>

One **proof receipt ID** feeds gates, checks, and shareable proof pages. **[Proof receipts](./platform/receipts-and-results)** defines what to save. **[Security and trust](./platform/security-and-trust)** defines visibility.

## 1. Create your account

1. Sign in at [`https://neus.network/hub`](https://neus.network/hub) with passkey or wallet.
2. Complete profile basics so you can see your **Apps**, **Credits**, and **Access keys**.

## 2. Run your first loop locally

Use **[Hosted Verify](./cookbook/auth-hosted-verify)** when the flow needs the hosted UI (passkey, OAuth, cross-device). For a first direct-sign SDK proof, follow **[Quickstart](./quickstart)**. **[Integration](./integration)** describes the check → verify → save → reuse loop.

## 3. Platform setup for production domains

For DNS domain proof, org OAuth, app linking, or sponsor grants at scale, use **[Platform setup](./platform/setup)** after the integration loop is clear.

## Before production

Three decisions: **create an app**, **choose who pays**, and **generate an access key** only when you need account-scoped API or MCP tools. Details for headers and limits live in **[Billing](./platform/billing)** and **[Platform setup](./platform/setup)**.

### Create an app

1. Open **[Apps](https://neus.network/profile?tab=apps)** in your NEUS account.
2. **Link app** or **Manage app**.
3. Enter your **app domain** (for example `acme.com`), set a **daily credit cap**, and finish the modal.

When registration succeeds, NEUS issues an **`appId`**. The SDK sends it as **`X-Neus-App`** on API traffic.

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: 'your-app-id', // same value sent as X-Neus-App
});
```

<Tip>
  **`appId`** tells NEUS **which product** made the request. It does **not** by itself move charges to your organization. When **you** pay for users, add **sponsor grants** and org wallet credits. See **Choose who pays** and **[Billing](./platform/billing)**.
</Tip>

### Choose who pays

**Default for shipped products:** **your product pays** via org-wallet credits and short-lived **sponsor grants** (`sponsorGrant` on the SDK → **`X-Sponsor-Grant`**). Users run Hosted Verify; they should not hit “buy credits” as a first step. **[Platform setup](./platform/setup)** has the full flow.

**Advanced / self-serve:** end users pay with **their** credits or **x402** when applicable. Do **not** send sponsor grants on those flows.

```javascript
const client = new NeusClient({
  appId: 'your-app-id',
  sponsorGrant: '<sponsor-grant-token>',
});
```

### Generate an access key for account-scoped tools

Use **[Access keys](https://neus.network/profile?tab=account)** when you need **`Authorization: Bearer`** on **`api.neus.network`**, scripts, or MCP tools such as **`neus_me`** and private reads. You can skip this until something returns **`auth_required`**. See **[MCP Auth](./mcp/auth)**.

Do **not** ship this key in browser bundles. End users complete interactive steps on **Hosted Verify**, not with your personal access key.

### Production readiness checklist

- [ ] App registered with a realistic daily cap
- [ ] Paying wallet has credits (yours or users', based on your model)
- [ ] Client sends **`appId`** and uses **`sponsorGrant`** only when you sponsor
- [ ] Team understands **[Billing](./platform/billing)** and **[x402](./platform/x402)**
- [ ] Verification receipts are stored and reused (see **[Proof receipts](./platform/receipts-and-results)**)
