## 1. Account

1. [Hub](https://neus.network/hub) - passkey or wallet.
2. Finish profile so **Apps**, **Credits**, and **Access keys** are available.

## 2. First loop

Browser-friendly verification to [Hosted Verify](./cookbook/auth-hosted-verify). First SDK wallet sign to [Quickstart](./quickstart). Loop to [Integration](./integration).

## 3. Org setup

DNS, org OAuth, apps to [Platform setup](./platform/setup) once the loop is clear.

## Launch checklist

**App**, **who pays**, **access key** (only for account-scoped API/MCP). [Billing](./platform/billing) | [Platform setup](./platform/setup)

### Create an app

1. [Apps](https://neus.network/profile?tab=apps) to **Link app** / **Manage app**.
2. **App domain** + **daily credit cap**.

You get an `appId`; pass it into the SDK so NEUS can attribute usage to your app.

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: 'your-app-id',
});
```

`appId` is attribution. Tie it to **your org billing** in app setup; use a **sponsor grant** on SDK calls when you want customer traffic to bill your balance - [Billing](./platform/billing).

### Choose who pays

**Typical product:** org credits plus a short-lived **`sponsorGrant`** on the SDK so customer sessions debit **your** balance, not the end user. [Platform setup](./platform/setup)

**Self-serve:** user credits or x402 (no sponsor).

```javascript
const client = new NeusClient({
  appId: 'your-app-id',
  sponsorGrant: '<sponsor-grant-token>',
});
```

### Access keys

[Access keys](https://neus.network/profile?tab=account) when a tool or script needs to act on **your** account (for example certain MCP tools). Create a key only after the product returns **`auth_required`**. [MCP Auth](./mcp/auth)

Never ship this key in the browser. Users use Hosted Verify, not your personal key.

### Checklist

- [ ] App + realistic daily cap
- [ ] Credits aligned to your payer model
- [ ] Billing matches your model: `appId` + `sponsorGrant` for org-paid customer traffic; otherwise the signer pays ([Billing](./platform/billing))
- [ ] Team read [Billing](./platform/billing) and [x402](./platform/x402)
- [ ] Verification IDs stored and reused ([Verification results](./platform/receipts-and-results))
