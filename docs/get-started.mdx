Pick your surface — app, assistant, agent, or backend. You sign in once to a **NEUS account**. Receipts travel with that account so every product does not restart verification from zero.

**Glossary:** **Check** is the product term for a trust requirement. In API requests the field is `verifierIds`. **Receipt ID** is the product term for a saved result; in API responses the field is `qHash`.

| You are building | Simplest path | Start |
| ---------------- | ------------- | ----- |
| **Gated content, marketplace, or checkout** | Publish a portal → `<VerifyGate>` with one `gateId` | [Quickstart](./quickstart) · [Create a portal](./platform/setup) |
| **Sign-in without building auth UI** | Hosted Verify (`intent=login`) — passkey / OAuth on NEUS | [Hosted Verify](./cookbook/auth-hosted-verify) · [Trust stack](./platform/trust-stack) |
| **An assistant in your IDE** | One command: `neus setup` | [Install NEUS](./install) |
| **Agents with limits before tools run** | Agent identity + permission receipts | [Agent verification flow](./agents/agent-verification-flow) |
| **A backend in any language** | HTTP API and server `gateCheck` | [API overview](./api/overview) |

**How billing works:** visitors complete checks on NEUS and keep the receipt; they can reuse it at your next gate when the checks match. You bill with credits (you pay by default) or charge visitors — [Pricing](./platform/pricing) · [Billing](./platform/billing).
