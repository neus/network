Use this checklist when you are shipping a product that relies on NEUS for verification, gating, or login.  
If you are experimenting with your own wallet, start with **[Quickstart](./quickstart)** and return here before production launch.

## 1. Create your account

1. Sign in to the **[NEUS platform](https://neus.network/hub)** with passkey or wallet.
2. Confirm the wallet that should own your integration and billing setup.

## 2. Register your integration and capture `appId`

Register the domain(s) your product will use, choose a daily credit cap, and complete the ownership signature step.

When registration succeeds, NEUS issues an **`appId`** for that integration. Send it as **`X-Neus-App`** on API traffic (the SDK maps this automatically).

Direct URL for the app registration surface:

`https://neus.network/profile?tab=apps`

SDK example:

```javascript
import { NeusClient } from '@neus/sdk';

const client = new NeusClient({
  appId: 'your-app-id', // same value sent as X-Neus-App
});
```

<Tip>
  **`appId`** is an attribution identifier (which product made the request).  
  It does **not** by itself move charges to your organization wallet. Use sponsor grants when you want to pay on behalf of users.
</Tip>

## 3. Fund the wallet that will pay

1. Keep credits on the wallet that should pay for your integration.
2. If users self-pay, they need their own credits for signed paid calls. See **[Billing](./platform/billing)**.

## 4. Decide your billing model (`sponsorGrant` if you pay)

If your product should pay verification costs for end users, issue short-lived sponsor grants and pass them to the SDK as **`sponsorGrant`** (maps to **`X-Sponsor-Grant`**).

If users should self-pay, do not send sponsor grants.

```javascript
const client = new NeusClient({
  appId: 'your-app-id',
  sponsorGrant: '<sponsor-grant-token>',
});
```

Grant issuance and caps: **[Partner setup](./cookbook/hub-integrator)** and **[Billing](./platform/billing)**.

## 5. Choose your integration path

| Option | When | Next step |
|---------|------|-----------|
| **Hosted Verify** | You want NEUS-hosted UI for login / verification | **[Hosted Verify](./cookbook/auth-hosted-verify)** |
| **Widgets** | React app, gates and badges | **[Widgets](./widgets/overview)** |
| **SDK** | Custom browser or Node flow | **[SDK overview](./sdks/overview)** → **[JavaScript](./sdks/javascript)** |
| **HTTP API** | Server-only | **[API overview](./api/overview)** |
| **MCP** | Cursor, Claude, agents | **[MCP overview](./mcp/overview)** |

Not sure? **[Choose an integration path](./choose-an-integration-path)**.

## 6. Production readiness checklist

- [ ] Integration domain is registered with a realistic daily cap
- [ ] Paying wallet has credits (yours or users', based on your model)
- [ ] Client sends **`appId`** and uses **`sponsorGrant`** only when sponsoring
- [ ] Team understands **[Billing](./platform/billing)** and **[x402](./platform/x402)** behavior
- [ ] Verification receipts are stored and reused in your product logic

## Without a linked app

Use **[Quickstart](./quickstart)** and **[Hosted Verify](./cookbook/auth-hosted-verify)** for early testing.  
Before launch, register your integration and add sponsor grants if your app will cover user costs.
