These samples implement the same flow: check for an existing receipt, verify if needed, save the `qHash`, reuse it on return visits. For production API usage, link your app and billing ([Get started](./get-started)).

**Default showcase:** [trust receipts showcase](https://github.com/neus/network/tree/main/examples/trust-receipts-showcase) (clone and `npm run dev`) maps real-world outcomes to verifiers. **Live product demo:** [neus.network/demo](https://neus.network/demo) — try flows in the browser, then use the same patterns from the open-source app.

For SDK setup, start with [Quickstart](./quickstart). Use [Hosted Verify](./cookbook/auth-hosted-verify) when NEUS should handle login, passkeys, wallets, OAuth, or gate checkout. Raw HTTP samples at the bottom of this page are **advanced**.

| Example | Use it for |
|------------|----------|
| [**trust-receipts-showcase** (default)](https://github.com/neus/network/tree/main/examples/trust-receipts-showcase) | Full **VerifyGate** + gate check: each card is a trust outcome and a verifier |
| [react-component](https://github.com/neus/network/tree/main/examples/react-component) | Minimal browser **`VerifyGate`** (Vite) |
| [nodejs-basic](https://github.com/neus/network/tree/main/examples/nodejs-basic) | **Advanced:** server key, sign + `gateCheck` |
| [curl-minimal](https://github.com/neus/network/tree/main/examples/curl-minimal) | **Advanced:** raw HTTP sign + submit |

## React: gate

```jsx
import { VerifyGate } from '@neus/sdk/widgets';

export default function ProtectedPage() {
  return (
    <VerifyGate
      appId="your-app-id"
      requiredVerifiers={['ownership-basic']}
    >
      <ProtectedContent />
    </VerifyGate>
  );
}
```

## Node: receipt

Use [Quickstart](./quickstart) for the normal server path: your app keeps login, the SDK calls NEUS from your backend. Use [nodejs-basic](https://github.com/neus/network/tree/main/examples/nodejs-basic) only when you need the lower-level SDK flow.

## HTTP (advanced)

```bash
curl -X POST https://api.neus.network/api/v1/verification/standardize \
  -H "Content-Type: application/json" \
  -d '{"walletAddress":"0x...","verifierIds":["ownership-basic"],"data":{"content":"Hello NEUS","owner":"0x..."},"signedTimestamp":1713312000000}'
# Advanced raw HTTP only: sign signerString, then POST /api/v1/verification with the same body + signature
```
